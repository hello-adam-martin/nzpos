    **Create `src/actions/orders/processPartialRefund.ts`:**

    Follow the exact pattern from `processRefund.ts` but with partial refund logic:

    ```
    'use server'
    import 'server-only'
    ```

    Import: `revalidatePath`, `PartialRefundSchema` from `@/schemas/refund`, `stripe` from `@/lib/stripe`, `createSupabaseAdminClient` from `@/lib/supabase/admin`, `createSupabaseServerClient` from `@/lib/supabase/server`, `buildCreditNote` from `@/lib/xero/buildInvoice`, `getAuthenticatedXeroClient` from `@/lib/xero/client`.

    Define `REFUNDABLE_STATUSES` set: `completed`, `partially_refunded`, `pending_pickup`, `ready`, `collected` (note: includes `partially_refunded` unlike old action).

    Define `calculateItemRefundCents(item: { line_total_cents: number; quantity: number }, quantityToRefund: number): number` — returns `Math.floor((quantityToRefund / item.quantity) * item.line_total_cents)`.

    Export async function `processPartialRefund(input: unknown)` returning `{ success: true; refundId: string } | { error: string }`:

    Transaction order (critical — follow exactly):
    1. Auth: `createSupabaseServerClient` -> `auth.getUser()` -> extract `store_id` from `app_metadata`
    2. Validate: `PartialRefundSchema.safeParse(input)` — return error if fails
    3. Fetch order: `adminClient.from('orders').select('*, order_items(*)').eq('id', orderId).eq('store_id', storeId).single()`
    4. Validate status: check `REFUNDABLE_STATUSES.has(order.status)` and `order.status !== 'refunded'`
    5. Fetch existing refund_items for this order to compute already-refunded quantities:
       ```
       adminClient.from('refund_items').select('order_item_id, quantity_refunded, refund_id')
         .in('refund_id', (await adminClient.from('refunds').select('id').eq('order_id', orderId)).data?.map(r => r.id) ?? [])
       ```
    6. Build a Map of `order_item_id -> totalAlreadyRefunded` from existing refund_items
    7. Validate each selected item: `quantityToRefund <= (item.quantity - alreadyRefundedQty)`. If any exceeds, return `{ error: 'Cannot refund more than remaining quantity for item: {product_name}' }`
    8. Calculate per-item refund cents using `calculateItemRefundCents`, sum for `totalRefundCents`
    9. Insert `refund` record into `refunds` table (pending state — this is the idempotency anchor):
       `{ order_id, store_id, reason, total_cents: totalRefundCents, created_by: user.id, customer_notified: false }`
    10. For online orders with `stripe_payment_intent_id`: call `stripe.refunds.create({ payment_intent: order.stripe_payment_intent_id, amount: totalRefundCents })`. On failure: delete the pending refund record, return error.
    11. If Stripe succeeded, update refund record with `stripe_refund_id`.
    12. Call `restore_stock` RPC for each selected item: `adminClient.rpc('restore_stock', { p_product_id: orderItem.product_id, p_quantity: quantityToRefund })`
    13. Insert `refund_items` records: one per selected item with `refund_id`, `order_item_id`, `quantity_refunded`, `line_total_refunded_cents`
    14. Determine new order status: re-query all refund_items for this order, compare total refunded qty per order_item_id against original quantities. If all fully refunded -> `'refunded'`, else -> `'partially_refunded'`.
    15. Update order status.
    16. Attempt Xero credit note (D-09 graceful failure):
        - Query `xero_sync_log` for matching `sync_date = order.created_at::date` and `status = 'success'`
        - If found, call `getAuthenticatedXeroClient(storeId)`, then `buildCreditNote(totalRefundCents, orderDate, xeroSettings, syncLog.xero_invoice_number)`, then `xero.accountingApi.createCreditNotes(tenantId, { creditNotes: [creditNote] })`
        - Wrap entire Xero block in try/catch — on any error, log warning but do NOT return error
    17. `revalidatePath('/admin/orders')`, `revalidatePath('/admin/reports')`, `revalidatePath('/admin/dashboard')`
    18. Return `{ success: true, refundId: refund.id }`
