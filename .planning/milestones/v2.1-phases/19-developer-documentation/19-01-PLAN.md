    Write docs/setup.md (DOC-01, per D-09, D-10, D-11, D-12):

    **Title:** "# Local Development Setup"
    **Intro:** "Get NZPOS running locally in under 20 minutes. This guide covers both local Supabase (recommended) and remote Supabase project setup."

    **Prerequisites section:**
    - Node.js 18+ (check with `node -v`)
    - Docker Desktop (required for local Supabase — `docker -v`)
    - Supabase CLI (`npx supabase --version` or install globally)
    - Stripe CLI (`stripe --version` — needed for webhook testing)
    - Git

    **Step 1: Clone and Install**
    ```bash
    git clone <repo-url>
    cd nzpos
    npm install
    ```

    **Step 2: Start Supabase (Primary Path — Local, per D-09)**
    ```bash
    npx supabase start
    ```
    Note: First run pulls Docker images (~5-10 minutes). Subsequent starts are fast (~10 seconds).
    After start, note the output values for `API URL`, `anon key`, and `service_role key`.

    **Step 2 (Alternative): Remote Supabase Project (per D-09)**
    - Create project at supabase.com
    - Run migrations: `npx supabase db push --linked`
    - Get credentials from dashboard > Settings > API

    **Step 3: Configure Environment**
    ```bash
    cp .env.example .env.local
    ```
    Fill in values from `supabase start` output. For full reference, see [Environment Variables](env-vars.md).
    Key values to set:
    - NEXT_PUBLIC_SUPABASE_URL → API URL from supabase start (default: http://127.0.0.1:54321)
    - NEXT_PUBLIC_SUPABASE_ANON_KEY → anon key from supabase start
    - SUPABASE_SERVICE_ROLE_KEY → service_role key from supabase start
    - STAFF_JWT_SECRET → generate with `openssl rand -base64 32`
    - ROOT_DOMAIN and NEXT_PUBLIC_ROOT_DOMAIN → `lvh.me:3000`
    - NEXT_PUBLIC_BASE_URL → `http://localhost:3000`

    **Step 4: Seed the Database**
    ```bash
    npx tsx supabase/seed.ts
    ```
    Creates: demo store ("mystore" slug), owner user, staff member, sample categories, and products.

    **Step 5: Start the Dev Server**
    ```bash
    npm run dev
    ```
    Visit:
    - Storefront: `http://mystore.lvh.me:3000`
    - Admin dashboard: `http://mystore.lvh.me:3000/admin`
    - POS: `http://mystore.lvh.me:3000/pos`
    - Super admin: `http://lvh.me:3000/super-admin`

    Note: `lvh.me` is a public wildcard DNS that resolves all subdomains to 127.0.0.1. Do NOT use `localhost` — middleware needs subdomain to resolve the tenant.

    **Step 6: Stripe Webhooks (per D-11)**
    Open two additional terminals:
    ```bash
    # Terminal 2 — Order webhooks
    stripe listen --forward-to localhost:3000/api/webhooks/stripe

    # Terminal 3 — Billing webhooks
    stripe listen --forward-to localhost:3000/api/webhooks/stripe/billing
    ```
    Copy the `whsec_...` values from each into `.env.local` as `STRIPE_WEBHOOK_SECRET` and `STRIPE_BILLING_WEBHOOK_SECRET`.

    **Step 7: Run Tests**
    ```bash
    npm run test          # Unit tests
    npm run test:coverage # With coverage report
    ```

    **Troubleshooting section (per D-12):**
    Format as H2 with individual H3 entries:

    **### RLS Blocking Queries**
    Symptom: "permission denied" errors on admin actions.
    Fix: Ensure `SUPABASE_SERVICE_ROLE_KEY` is set in `.env.local`. Server Actions use `createSupabaseAdminClient()` (service role) for mutations.

    **### Missing Environment Variables**
    Symptom: Runtime errors about undefined env vars.
    Fix: Compare `.env.local` against `.env.example`. All 24 variables must be set. See [env-vars.md](env-vars.md) for details.

    **### Subdomain Routing on Localhost**
    Symptom: 404s on /admin or /pos, store not resolving.
    Fix: Use `{slug}.lvh.me:3000` not `localhost:3000`. Middleware checks the host header for subdomain extraction. `localhost` is treated as root domain.

    **### Supabase Migration Issues**
    Symptom: Schema errors after pulling new code.
    Fix: Run `npx supabase db reset && npx tsx supabase/seed.ts`. This re-applies all 23 migrations and re-seeds.

    **### supabase db reset Wipes Seed Data**
    Symptom: Empty database after reset.
    Fix: `supabase db reset` runs `seed.sql` not `seed.ts`. Always follow with `npx tsx supabase/seed.ts`.

    **### JWT Claims Not Updated After Provisioning**
    Symptom: Owner can log in but /admin redirects to unauthorized.
    Fix: Middleware auto-refreshes sessions when `app_metadata.role` is missing. Try hard-refreshing the browser. If persisted, sign out and back in.
