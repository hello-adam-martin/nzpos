    **CSP Policy (single policy for all surfaces per D-08):**
    Add the following CSP construction AFTER the existing auth/tenant routing logic but BEFORE returning the response. Set the header on the NextResponse object that the middleware returns:

    ```typescript
    // Security headers (Phase 17 — SEC-12)
    const cspDirectives = [
      "default-src 'self'",
      "script-src 'self' 'unsafe-inline' https://js.stripe.com",
      "style-src 'self' 'unsafe-inline'",
      "img-src 'self' data: blob: https://*.supabase.co",
      "connect-src 'self' https://*.supabase.co https://api.stripe.com wss://*.supabase.co",
      "frame-src https://js.stripe.com https://hooks.stripe.com",
      "font-src 'self'",
      "object-src 'none'",
      "base-uri 'self'",
      "form-action 'self'",
    ].join('; ')

    // Report-Only first (D-06) — switch to enforcing after validation
    response.headers.set('Content-Security-Policy-Report-Only', cspDirectives)
    ```

    Also add these standard security headers:
    ```typescript
    response.headers.set('X-Content-Type-Options', 'nosniff')
    response.headers.set('X-Frame-Options', 'DENY')
    response.headers.set('Referrer-Policy', 'strict-origin-when-cross-origin')
    ```

    Do NOT add `Strict-Transport-Security` — Vercel handles this at the edge.

    **Implementation details:**
    - Find ALL return points in the middleware that return a NextResponse
    - Create a helper function `addSecurityHeaders(response: NextResponse): NextResponse` that sets all headers
    - Call this helper before every `return response` statement
    - This ensures CSP applies to storefront, POS, admin, and super admin surfaces (per D-08)
    - If middleware has early returns (e.g., for static assets, API routes), ensure those ALSO get security headers OR are explicitly excluded with a comment explaining why

    **Verify Stripe compatibility:**
    - `script-src` includes `https://js.stripe.com` for Stripe.js
    - `frame-src` includes `https://js.stripe.com` and `https://hooks.stripe.com` for Stripe iframes
    - `connect-src` includes `https://api.stripe.com` for API calls
    - `style-src` includes `'unsafe-inline'` for Tailwind's inline styles
    - `img-src` includes `https://*.supabase.co` for product images from Supabase Storage
    - `connect-src` includes `wss://*.supabase.co` for any WebSocket connections (Supabase auth refresh)
