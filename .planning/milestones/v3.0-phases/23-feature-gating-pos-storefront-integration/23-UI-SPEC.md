## Spacing Scale

Declared values (all multiples of 4, verified in globals.css `:root`):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline element gaps |
| sm | 8px | Compact list item spacing, tight label groups |
| md | 16px | Default element spacing, card inner padding |
| lg | 24px | Card padding, section gaps within a view |
| xl | 32px | Major layout gaps, between card groups |
| 2xl | 48px | Section breaks, upgrade wall content spacing |
| 3xl | 64px | Page-level vertical rhythm (rarely needed in admin) |

Exceptions:

- POS product cards: minimum touch target 44x44px (DESIGN.md POS-specific rule)
- Upgrade wall CTA button: minimum height 44px (matches existing amber button in UpgradePrompt.tsx)
- Stock badge text within ProductCard: uses `mt-1` (4px Tailwind) — intentional tight coupling to price line

## Interaction Contract

### 1. Upgrade Wall — `/admin/inventory` for ungated stores

**Trigger:** User without `has_inventory === true` navigates to `/admin/inventory` (direct URL or bookmark).

**Render:** Server Component renders `<InventoryUpgradeWall />` instead of inventory page content. No redirect. Page URL stays `/admin/inventory`.

**Layout:**
```
[Admin layout shell — sidebar, header]
  └── [Content area — vertically centered within main]
        └── [Card — bg-surface, border, rounded-lg, p-6, max-w-lg, mx-auto, mt-xl]
              ├── [Lock icon — 32px, text-text-light, mb-md]
              ├── [Headline — text-lg, font-semibold]
              ├── [Body — text-base, text-text-muted, mb-lg]
              └── [CTA button — amber bg, white text, sm font-semibold, px-4 py-2.5, rounded-md, full-width]
```

**Hover:** CTA button `hover:opacity-90` at 150ms ease-out (matches existing UpgradePrompt.tsx pattern).

**Accessibility:** Card has no `role`. CTA is an `<a>` tag (navigation, not action). No `aria-label` needed — button text is descriptive.

### 2. Success Banner — Billing page after inventory subscription

**Trigger:** `?subscribed=inventory` in URL when BillingClient mounts.

**Render:** Inline success banner below the billing page header. Not a fixed-position toast overlay (avoids portal complexity — matches existing inline `<p role="alert">` error pattern in BillingClient).

**Layout:**
```
[Success banner — bg-[#ECFDF5], border border-[#059669]/20, rounded-md, p-3, mb-xl]
  └── [Text — text-sm, font-semibold, text-[#059669], role="alert"]
        "Inventory add-on activated! Your store now has full stock management."
```

**Dismiss behavior:** `useEffect` with `router.replace` strips `?subscribed` param after 4000ms timeout, removing the banner on next render. Banner does not re-appear on page refresh (URL param cleared).

**Accessibility:** `role="alert"` announces to screen readers on mount. No close button needed (auto-dismisses).

### 3. POS Stock Badges — Visual states (existing, must match spec)

**Condition:** Only rendered when `hasInventory === true` AND `product_type !== 'service'`.

| State | Text | Text Color | Weight |
|-------|------|-----------|--------|
| In-stock (quantity > reorder_threshold) | `{N} in stock` | `--color-text-light` (#A8A29E) | 400 |
| Low-stock (0 < quantity ≤ reorder_threshold) | `{N} in stock` | `--color-warning` (#D97706) | 400 |
| Out-of-stock (quantity === 0) | `Out of stock` | `--color-error` (#DC2626) | 600 (semibold) |

Note: Low-stock threshold uses `product.reorder_threshold` (default 5) — Decision D-07.

### 4. POS Cart Block — Out-of-stock product interaction

**Staff path:** Tapping an out-of-stock card does nothing (card `isDisabled = true`, `opacity-50`, `cursor-not-allowed`). Keyboard: `tabIndex={-1}`, not focusable.

**Owner path:** Tapping an out-of-stock card adds it to cart directly (no dialog) per existing `POSClientShell.handleAddToCart` owner branch. This is already implemented — no visual change needed.

Note: RESEARCH.md confirms `OutOfStockDialog` exists and is already wired. The owner override (D-09) is already implemented in `POSClientShell` — staff gets `OutOfStockDialog`, owner bypasses it. This spec confirms the existing behavior; no redesign.

### 5. Storefront Sold Out — Product card and button states (existing, must match spec)

**Sold-out badge on StoreProductCard:** Amber-tinted "Sold Out" overlay or badge (existing implementation — do not change).

**Add to Cart button disabled:** `opacity-50 cursor-not-allowed`, button text changes to "Sold Out", `disabled` attribute set.

**Condition:** Only applies when `hasInventory === true` AND `product_type !== 'service'` AND `stock_quantity <= 0`.

### 6. Super Admin Toggle — Inventory override (existing pattern, no new visual work)

Toggle switch on `/super-admin/tenants/[id]` page, rendered by existing `PlanOverrideRow` component via ADDONS array iteration. No new visual component. The fix is purely a data query change (add `has_inventory*` columns to the select).
