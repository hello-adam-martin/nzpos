    - Full-width layout (no max-width)

      - Left: stocktake date/scope (20px Satoshi 700 = `text-xl font-bold font-display`)
      - Right: "Commit stocktake" button (amber `bg-amber text-white rounded-md px-4 py-2 font-bold`) + "Discard stocktake" button (ghost, `text-muted hover:text-error`)
      - "Commit stocktake" disabled (`opacity-50 cursor-not-allowed`) until at least one line has counted_quantity !== null
      - Only show action buttons when session status is 'in_progress'
    - **Tab navigation:** "Count" and "Review" tabs using TabButton pattern (URL-driven or state-driven)
    - **Count tab:** renders `StocktakeCountTab`
    - **Review tab:** renders `StocktakeReviewTab`

    **Commit stocktake (per UI-SPEC Interaction States):**
    - Clicking "Commit stocktake" shows inline confirmation strip below the button (not a modal)
    - Strip background: `bg-warning/10 border border-warning rounded-md p-4`
    - Copy: "Commit this stocktake? This will update {N} product quantities. This cannot be undone."
    - N = count of lines where counted_quantity IS NOT NULL
    - Buttons: "Yes, commit" (amber) + "Keep reviewing" (ghost)
    - Strip appears with 150ms ease-out height expand animation
    - Strip has `role="alert"` for accessibility
    - On "Yes, commit": call `commitStocktake({ session_id })`, on success redirect to `/admin/inventory?tab=stocktakes`
    - On error: show error copy "Commit failed. No stock quantities were changed. Try again or contact support if this continues." in `text-error`

    **Discard stocktake (per UI-SPEC Interaction States):**
    - Clicking "Discard stocktake" opens a modal
    - Modal: `bg-card rounded-lg shadow-lg p-6` (p-lg = 24px), max-width 400px, centered with backdrop
    - Title: "Discard stocktake?" (20px Satoshi 700)
    - Body: "All counted quantities will be lost. This cannot be undone." (14px DM Sans muted)
    - Buttons: "Discard stocktake" (`bg-error text-white rounded-md`) + "Keep counting" (ghost)
    - Focus trap while open, focus returns to trigger on close
    - On confirm: call `discardStocktakeSession({ session_id })`, redirect to `/admin/inventory?tab=stocktakes`

    **3. Create `src/components/admin/inventory/StocktakeCountTab.tsx`** ('use client'):
    - Receives `lines` array and `sessionStatus` as props
    - **Search/filter bar:** full-width search input (filters by product name, SKU, or barcode) + `BarcodeScannerButton` component
    - **Scrollable table:** Product, SKU, Barcode, Counted Qty columns
    - System quantity column: HIDDEN (anti-anchoring per D-07)
    - Count input per row: `<input type="number" min="0">` — 64px wide, Geist Mono 16px (`font-mono text-base`), center-aligned, `border-border rounded-md px-2`
    - Empty input shows placeholder "—" (not 0) to distinguish not-counted from counted-as-zero
    - All inputs disabled when session status !== 'in_progress'
    - `aria-label="Counted quantity for {product name}"` on each input

    **Auto-save (per D-08 and UI-SPEC Interaction States):**
    - On input change: update local state immediately (optimistic)
    - Debounce 800ms, then call `updateStocktakeLine({ line_id, counted_quantity })` via `useTransition`
    - Save indicator in top-right corner: "Saving..." (animated ellipsis via CSS, `text-muted text-sm`) -> "Saved" (fades out after 2000ms) -> on error: "Save failed — check connection" in `text-error` (persists until next success)
    - Each input uses a ref map keyed by product_id for barcode auto-focus

    **Barcode scanner integration (TAKE-05):**
    - Render `BarcodeScannerButton` in the search bar area
    - On scan result, call `lookupBarcode(barcode)` to get product
    - If product found and exists in lines: scroll to matching row, highlight row with `bg-amber/10 border-l-4 border-amber` (150ms ease-out transition), auto-focus the count input
    - If product not found in stocktake lines: show brief toast/message "Product not in this stocktake"
    - Auto-focus uses a `Map<string, HTMLInputElement>` ref keyed by product_id (per RESEARCH.md Pattern 6)

    **4. Create `src/components/admin/inventory/StocktakeReviewTab.tsx`** ('use client'):
    - Receives `lines` array as props
    - **Table columns:** Product, System Qty, Counted Qty, Variance
    - System quantity: VISIBLE here (per D-07 — only shown on Review tab)
    - All quantity columns: Geist Mono (`font-mono`), `font-feature-settings: 'tnum' 1`
    - Variance cell: `counted_quantity - system_snapshot_quantity`
      - Positive: "+N" in `text-success` (#059669)
      - Negative: "-N" in `text-error` (#DC2626)
      - Zero: "0" in `text-muted` (#78716C)
      - Use `aria-label="{N} ({shortage|surplus})"` for screen readers
    - **Filter toggle:** "Show all" / "Show variances only" pill toggle (default: "Show variances only")
      - Active pill: `bg-navy text-white rounded-full px-3 py-1`
      - Inactive pill: `bg-surface text-text rounded-full px-3 py-1`
      - "Show variances only" filters to lines where `counted_quantity !== null && counted_quantity !== system_snapshot_quantity`
    - Lines where `counted_quantity` is null show "Not counted" in muted text in the Counted Qty column
    - Show summary at top: "X products counted, Y with variance, Z not counted"
