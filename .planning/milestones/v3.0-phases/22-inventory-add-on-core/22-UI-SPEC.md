## Color

All values resolved from `globals.css` @theme tokens.

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) | `--color-bg` | #FAFAF9 | Page background (warm stone white) |
| Secondary (30%) | `--color-surface` + `--color-card` | #F5F5F4 / #FFFFFF | Table alternating rows (surface/card), sidebar, drawer background |
| Accent (10%) | `--color-amber` | #E67E22 | Reserved-for list below |
| Destructive | `--color-error` | #DC2626 | Discard stocktake session button + confirmation modal |

**Accent (#E67E22) reserved for these specific elements only:**
1. "Adjust stock" button in product edit view (primary action trigger)
2. "Start stocktake" call-to-action button on Stocktakes tab empty state
3. "Commit stocktake" button on StocktakeReviewTab (final commit action)
4. Low-stock badge background on Stock Levels tab (stock <= reorder_threshold, not zero)

**Semantic colors (not accent):**
- Out-of-stock quantity text: `--color-error` (#DC2626) — matches existing `StockLevelsTable` pattern (`text-error font-bold`)
- Low-stock quantity text: `--color-warning` (#D97706) — matches existing `StockLevelsTable` pattern (`text-warning font-bold`)
- Positive variance (count > system): `--color-success` (#059669)
- Negative variance (count < system): `--color-error` (#DC2626)
- Zero variance: `--color-text-muted` (#78716C)
- In-progress stocktake status badge: `--color-info` background tint (#3B82F6 at 10% opacity, blue text)
- Committed stocktake status badge: `--color-success` background tint (#059669 at 10% opacity, green text)
- Discarded stocktake status badge: neutral `--color-surface`, `--color-text-muted` text

Source: DESIGN.md color section + existing `StockLevelsTable.tsx` stock color logic + `LowStockAlertList.tsx` semantic color patterns.

## Component Inventory

New components required for this phase, with their key visual contracts:

### `/admin/inventory` page — InventoryPageClient

- Three tabs: "Stock Levels", "Stocktakes", "History"
- Tab navigation: reuse `TabButton` pattern from `ReportsPageClient` verbatim
  - Active: `border-navy text-navy border-b-2`
  - Inactive: `border-transparent text-muted hover:text-primary hover:border-border`
  - Container: `flex gap-2 border-b border-border pb-0`
- Tabs are URL-driven via `?tab=stock|stocktakes|history` searchParam
- **Focal point:** Primary visual anchor on page load is the table header row (`bg-navy text-white`). Low-stock rows with amber left border (`border-l-4 border-amber`) serve as secondary focal points drawing the owner's eye to products needing attention.

### StockLevelsTab

- Reuse `StockLevelsTable` from reports as starting pattern — extend with "Adjust stock" inline button per row
- Table header: navy background (`bg-navy text-white`), 14px bold DM Sans
- Table rows: alternating `bg-card` / `bg-surface`, 14px DM Sans
- Stock column: Geist Mono, semantic color (error/warning/primary per existing logic)
- "Adjust stock" per row: ghost button (transparent, `border-border`, 14px bold DM Sans), appears on row hover
- Low-stock rows: amber left border `border-l-4 border-amber` to draw eye without disrupting row layout

### StockAdjustDrawer

- Matches `ProductFormDrawer` layout: fixed right-side overlay, 480px wide, full height
- Backdrop: `bg-navy/40` (40% opacity navy)
- Drawer background: `bg-card` (#FFFFFF)
- Header: product name (20px Satoshi 700), current stock quantity (14px Geist Mono, muted)
- Toggle between "Adjust by" and "Set to" modes:
  - Toggle control: two pill buttons side by side, active pill `bg-navy text-white`, inactive `bg-surface text-text border-border`
  - Toggle container: `rounded-full border border-border` wrapper
- Quantity input: large (48px height), centered number, Geist Mono 16px, `border-border rounded-md`
  - "Adjust by" mode: shows `+` / `−` stepper buttons flanking the input
  - Stepper buttons: 44x44px touch target, `bg-surface hover:bg-border`
- Reason code: `<select>` styled to match existing admin selects — `border-border rounded-md px-md py-sm text-sm DM Sans`
- Notes: `<textarea>` rows=3, same border/radius pattern
- Preview row: shows computed result before save — "New stock: 47" in 14px Geist Mono bold
- Footer: "Save adjustment" primary button (navy bg, white text) + "Close without saving" ghost link
- Error inline below input: 14px DM Sans, `--color-error` text

### StocktakeSessionList (Stocktakes tab)

- Table with columns: Started, Scope, Products, Status, Action
- Status badges: pill shape (`rounded-full px-sm py-xs text-xs font-bold DM Sans`)
  - In progress: info tint (blue)
  - Committed: success tint (green)
  - Discarded: neutral (surface)
- "Continue" action for in-progress: amber text link
- "View" action for committed/discarded: muted text link
- Empty state: icon + heading + amber CTA button (see Copywriting)

### StocktakeCountPage (`/admin/inventory/stocktake/[id]`)

- Full-width layout (no max-width constraint — admin is full-width per DESIGN.md)
- Top bar: stocktake name/date (20px Satoshi 700) + "Commit stocktake" button (amber, right-aligned) + "Discard stocktake" button (ghost destructive, muted text, shows error color on hover)
- "Commit stocktake" disabled until at least one line has a counted quantity
- Tab navigation: "Count" and "Review" tabs (same TabButton pattern)
- Search/filter bar: full-width search input + barcode scanner button (matches `BarcodeScannerButton` pattern)

### StocktakeCountTab

- Scrollable table: Product, SKU, Barcode, Counted Qty
- System quantity column: hidden (anti-anchoring per D-07)
- Count input: `<input type="number" min="0">` — 64px wide, Geist Mono 16px, center-aligned, `border-border rounded-md px-sm`
- Auto-focused row (post-barcode scan): highlighted with `bg-amber/10 border-l-4 border-amber` row highlight, input auto-focused
- Auto-save indicator: subtle "Saving..." / "Saved" state in top-right corner — 14px DM Sans muted, fades after 2s. Debounce: 800ms after last keystroke
- Empty count input shows placeholder "—" (not 0) to distinguish "not yet counted" from "counted as zero"

### StocktakeReviewTab

- Table columns: Product, System Qty, Counted Qty, Variance, Action
- System quantity: visible here (matches D-07 — only shown on Review tab)
- Variance cell: Geist Mono, semantic color (success/error/muted per +/-/0)
- Positive variance: "+5" in `--color-success`
- Negative variance: "-3" in `--color-error`
- Zero variance: "0" in `--color-text-muted`
- Products with zero variance can be filtered out via "Show all / Show variances only" toggle (toggle pill, default "Show variances only" for focus)

### InventoryHistoryTab

- Filter bar: Product picker (search-enabled select) + date range (two date inputs) + reason code dropdown. All filters in a single horizontal row with "Clear filters" ghost link.
- Table columns: Date, Product, Reason, Change, New Total, Notes, User
- Change column: Geist Mono, `+5` in success green / `-3` in error red (matches variance color logic)
- Reason code displayed as human-readable label (not raw enum key): e.g. "Received stock", "Damaged", "Theft / shrinkage", "Correction", "Return to supplier", "Other", "Sale", "Refund", "Stocktake"
- Pagination: 50 rows per page, simple prev/next pagination controls at table bottom

## Interaction States

### Auto-save (Stocktake count entry)

- Debounce: 800ms after last keystroke
- Saving indicator: 14px DM Sans, `--color-text-muted`, "Saving..." with animated ellipsis (CSS animation, 150ms per dot)
- Success: "Saved" replaces "Saving...", fades out after 2000ms
- Error: "Save failed — check connection" in `--color-error`, persists until next successful save

### Commit stocktake confirmation

- Confirm via an inline confirmation strip (not a modal) that appears below the Commit button:
  - "Commit this stocktake? This will update {N} product quantities. This cannot be undone."
  - Two buttons: "Yes, commit" (amber) and "Keep reviewing" (ghost)
  - Strip background: `--color-warning` at 10% opacity, `border-warning` border
- Rationale: modal is heavier UX; inline strip matches admin's preference for in-context confirmation

### Discard stocktake

- Discard is destructive — requires modal confirmation
- Modal: `bg-card rounded-lg shadow-lg p-lg`, max-width 400px, centered
- Title: "Discard stocktake?" 20px Satoshi 700
- Body: "All counted quantities will be lost. This cannot be undone." 14px DM Sans muted
- Buttons: "Discard stocktake" (`bg-error text-white rounded-md`) + "Keep counting" (ghost)

### Adjust stock — save state

- "Save adjustment" button shows spinner (SVG animate-spin) during submission
- On success: drawer closes, parent row stock quantity updates optimistically
- On error: inline error below the form, drawer stays open

## Motion

Matches DESIGN.md motion rules:

| Element | Duration | Easing | Notes |
|---------|----------|--------|-------|
| Drawer open/close | 250ms | ease-out open / ease-in close | Slides in from right. Per DESIGN.md "medium: 250ms (modal open/close)" |
| Row highlight (barcode focus) | 150ms | ease-out | Short per DESIGN.md "short: 150ms (hover states)" |
| Auto-save indicator fade-out | 150ms | ease-in | Fade-out only, no fade-in |
| Confirm strip appear | 150ms | ease-out | Height expand from 0 |
| Button spinner | continuous | linear | SVG rotate, no easing needed |
| Hover states on table rows | 100ms | ease-out | Per DESIGN.md "micro: 50-100ms" |

No animations longer than 250ms in the admin context (DESIGN.md: "POS rule: Nothing in the POS checkout path should animate longer than 150ms" — admin extends this conservatively to 250ms max).

## Accessibility Notes

- All form inputs have associated `<label>` elements (not aria-label only)
- Stock quantity inputs: `aria-label="Counted quantity for {product name}"` on StocktakeCountTab rows (product name is not adjacent to input in a strict label sense)
- Variance column: screen reader reads "minus 3" not "-3" — use `aria-label="-3 (shortage)"` on variance cells
- Drawer: `role="dialog" aria-modal="true" aria-labelledby` pointing to drawer heading
- Confirm strip: `role="alert"` so screen reader announces appearance
- Discard modal: focus trap while open, focus returns to "Discard stocktake" trigger on close
