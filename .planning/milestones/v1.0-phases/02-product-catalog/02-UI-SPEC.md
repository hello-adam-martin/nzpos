## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tight inline padding, badge padding vertical |
| sm | 8px | Input padding vertical, chip gaps |
| md | 16px | Card padding, field spacing, default element gap |
| lg | 24px | Section padding, panel gaps |
| xl | 32px | Layout column gaps, form section spacing |
| 2xl | 48px | Page-level top/bottom padding |
| 3xl | 64px | Large modal max-height offsets |

Exceptions:
- Table row height: minimum 48px (comfortable click/tap target for owner on desktop + iPad)
- Image thumbnail in table: 40x40px (fixed size, centered in 48px row)
- Category sidebar: fixed 240px wide (matches DESIGN.md admin layout spec)
- CSV column mapper drag handle: 44x44px touch target (admin can be used on iPad)

Source: DESIGN.md spacing scale + POS touch target spec. Admin density is "comfortable" per DESIGN.md. 2xs/2px token removed — 2 is not a multiple of 4; badge vertical padding uses xs (4px).

- Caption/helper text shares the 14px (sm) size at weight 400 to distinguish from label/UI (weight 600).

## Interaction Contracts

### Table Sorting
- Click column header to sort ascending. Click again for descending. Third click clears sort (returns to default: name A–Z).
- Sort indicator: chevron up/down icon in column header. Navy color.
- Only sortable columns: name, SKU, price, stock quantity.

### Search + Filter
- Search debounce: 300ms after last keystroke.
- Filters and search combine with AND logic.
- Active filters shown as dismissible chips below the search bar.
- "Clear all filters" text link appears when any filter is active.

### Product Form Drawer
- Opens on "Add Product" or row click (edit mode).
- Slides in from right, 480px width on desktop. Full-screen on viewport < 640px.
- Backdrop (rgba black 40%) covers main content. Click backdrop to cancel (if no unsaved changes).
- Unsaved changes: clicking backdrop or pressing Escape shows inline confirmation: "You have unsaved changes. Discard?" with "Keep editing" and "Discard" buttons.
- Form validation runs on submit (not on blur). Error messages appear below each field in 14px / caption style / error red.
- Successful save: drawer closes, table row updates immediately (optimistic) then re-validates from server.

### Image Upload
- Accept: JPEG, PNG, WebP. Max client-side check: 10MB (server enforces, but client shows early error for UX).
- After selection: image preview replaces placeholder at 200x200px, cropped center (object-cover).
- Upload progress: subtle progress bar below image picker. Not a blocking spinner.
- Upload error: inline error below picker, "Upload failed — try again."

### CSV Import
- Three-step flow: Upload → Map Columns → Preview & Commit.
- Step indicator at top (1 / 2 / 3, horizontal steps).
- "Back" ghost button on steps 2 and 3.
- Preview loads asynchronously. Loading state: skeleton rows in preview table.
- Commit is final. No undo. Confirmation copy: "This will add {N} products to your catalog. You cannot undo this action."

### Category Drag-and-Drop
- Library: `@dnd-kit/core` + `@dnd-kit/sortable` (per Claude's discretion from CONTEXT.md).
- Drag handle shows grab cursor. Item lifts with sm shadow on drag start.
- Drop target: 2px navy line between items shows drop position.
- Reorder saves immediately on drop (Server Action, optimistic update).
- Keyboard-accessible: Spacebar to grab, arrow keys to move, Spacebar to drop, Escape to cancel.

### Deactivate Product
- "Deactivate Product" button in edit drawer (ghost, error color).
- Confirmation dialog (modal): heading "Deactivate [Product Name]?", body "This product will no longer appear in the POS or online store.", two buttons: "Keep Product Active" (ghost) and "Deactivate" (error/red background).
- Deactivated product row: reduced opacity (60%) in table. Status badge changes to "Inactive".

### Delete Category
- Only available if category has 0 products. If category has products, delete button is disabled with tooltip: "Move or remove products first."
- If 0 products: confirmation dialog: "Delete [Category Name]?" / "This cannot be undone." / "Keep Category" (ghost) + "Delete" (red).

## States Reference

Each interactive element must implement these states:

| Element | States Required |
|---------|----------------|
| Primary button (navy) | default, hover (navy-light), active (navy-dark), disabled (40% opacity), loading (spinner replaces label) |
| Accent button (amber) | default, hover (amber-hover #D35400), active (darker), disabled (40% opacity) |
| Ghost button | default, hover (surface bg), active (border-light), disabled (40% opacity) |
| Text input | default (border), focus (navy border 2px ring), error (error border + message below), disabled (surface bg, muted text) |
| Table row | default, hover (surface bg #F5F5F4), selected (navy-dark left border 2px + surface tint) |
| Category row | default, hover (surface), dragging (elevated shadow, 95% opacity) |
| Status badge | see ProductStatusBadge above |
| Toggle | off (border + surface), on (navy bg + white dot) |
