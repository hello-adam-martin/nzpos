  <action>
    Run `npx next build` and `npx vitest run` to verify everything compiles and tests pass.

    Review and fix these known edge cases:

    1. **Empty cart pay button:** Verify PayButton is disabled when `cart.items.length === 0 || cart.paymentMethod === null`. If not, fix.

    2. **Discount on removed item:** If a discount sheet is open for an item that gets removed, close the sheet. Check `discountTarget` still in `cart.items` when rendering DiscountSheet.

    3. **Double-tap prevention:** Ensure EFTPOS confirm buttons and Pay button set `isProcessing` to true immediately to prevent double-taps. Add `disabled={isProcessing}` to all payment action buttons.

    4. **Cash entry validation:** Ensure cash tendered input rejects negative values, non-numeric input, and extremely large values. Validate before calling handleCompleteSale.

    5. **Split payment remainder to EFTPOS:** Verify the flow: enter cash -> tap split -> EFTPOS confirm screen shows ONLY the remaining amount (not full total). Verify `handleCompleteSale` sends correct `payment_method: 'split'`.

    6. **Sale void cleanup:** When EFTPOS NO is tapped, verify cart fully resets (items cleared, payment method null, phase back to idle).

    7. **Focus trap on EFTPOS screen:** Verify Tab key cannot leave the EFTPOS confirmation dialog. Implement with a `useEffect` that stores previously focused element and restores on unmount.

    8. **Product image fallback:** Verify ProductCard gracefully handles `image_url === null` with a colored placeholder.

    9. **Sale summary data preservation:** When `SALE_COMPLETE` is dispatched, the cart items are about to be cleared by `NEW_SALE`. Ensure SaleSummaryScreen receives the items BEFORE the reducer clears them. The reducer should keep items in state during `sale_complete` phase — they are only cleared on `NEW_SALE`.

    10. **Motion compliance:** Verify all animations are <= 150ms per UI-SPEC POS rule. Check DiscountSheet slide (150ms), EftposConfirm fade (150ms), cart item add/remove (100ms if implemented, else instant).

    Fix any issues found. Run `npx next build` and `npx vitest run` after fixes.
