**Implementation steps (all within the exported function):**

1. **Verify owner auth** (admin-only, NOT staff per D-13):
   ```typescript
   const supabase = await createSupabaseServerClient()
   const { data: { user } } = await supabase.auth.getUser()
   if (!user) return { error: 'Not authenticated' }
   const storeId = user.app_metadata?.store_id as string | undefined
   if (!storeId) return { error: 'No store context' }
   ```

2. **Validate input with Zod:**
   ```typescript
   const parsed = RefundSchema.safeParse(input)
   if (!parsed.success) return { error: 'Invalid input' }
   const { orderId, reason, restoreStock } = parsed.data
   ```

3. **Fetch order with items** (admin client for RLS bypass):
   ```typescript
   const adminClient = createSupabaseAdminClient()
   const { data: order } = await adminClient
     .from('orders')
     .select('*, order_items(*)')
     .eq('id', orderId)
     .eq('store_id', storeId)
     .single()
   ```

4. **Validate refundable state:**
   - If `!order` -> return `{ error: 'Order not found' }`
   - If `order.status === 'refunded'` -> return `{ error: 'Order already refunded' }`
   - If `order.status` not in `['completed', 'pending_pickup', 'ready', 'collected']` -> return `{ error: 'Order cannot be refunded in current status' }`

5. **Stripe refund for online orders** per D-15, Pitfall #6:
   ```typescript
   if (order.channel === 'online' && order.stripe_payment_intent_id) {
     try {
       const refund = await stripe.refunds.create({
         payment_intent: order.stripe_payment_intent_id,
       })
       if (refund.status !== 'succeeded' && refund.status !== 'pending') {
         return { error: 'Stripe refund failed. Check your Stripe dashboard and try again.' }
       }
     } catch (err) {
       return { error: 'Stripe refund failed. Check your Stripe dashboard and try again.' }
     }
   } else if (order.channel === 'online' && !order.stripe_payment_intent_id) {
     return { error: 'Cannot refund — Stripe payment ID not found. Check order in Stripe dashboard.' }
   }
   ```

6. **Update order status to refunded:**
   ```typescript
   await adminClient
     .from('orders')
     .update({ status: 'refunded', notes: reason })
     .eq('id', orderId)
   ```

7. **Restore stock if requested** per D-14:
   Read-then-write pattern (safe for v1 single-operator per Research recommendation):
   ```typescript
   if (restoreStock && order.order_items) {
     for (const item of order.order_items) {
       const { data: product } = await adminClient
         .from('products')
         .select('stock_quantity')
         .eq('id', item.product_id)
         .single()
       if (product) {
         await adminClient
           .from('products')
           .update({ stock_quantity: product.stock_quantity + item.quantity })
           .eq('id', item.product_id)
       }
     }
   }
   ```

8. **Revalidate paths and return:**
   ```typescript
   revalidatePath('/admin/orders')
   revalidatePath('/admin/reports')
   revalidatePath('/admin/dashboard')
   return { success: true }
   ```

Return type: `Promise<{ success: true } | { error: string }>`
