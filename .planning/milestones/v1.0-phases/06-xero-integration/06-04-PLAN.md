**vercel.json** — Cron job configuration:
```json
{
  "crons": [
    {
      "path": "/api/cron/xero-sync",
      "schedule": "0 13 * * *"
    }
  ]
}
```
Note: `0 13 * * *` = 1:00 PM UTC = ~2:00 AM NZST (UTC+13 summer) / ~1:00 AM NZST (UTC+12 winter). Both are overnight, before business opening. Per D-08.

**src/app/api/cron/xero-sync/route.ts** — Automated daily sync with retry:
- Export `GET` handler.
- Verify `CRON_SECRET`: `const authHeader = req.headers.get('Authorization'); if (authHeader !== 'Bearer ' + process.env.CRON_SECRET) return new Response('Unauthorized', { status: 401 })`.
- Query all connected stores: use admin client to get all `xero_connections` where `status = 'connected'`.
- For each connected store: call `executeDailySyncWithRetry(storeId)` (NOT `executeDailySync`). This implements D-09 retry logic — up to 3 attempts with exponential backoff (1min, 5min, 15min). The retry logic is in the sync module; the cron route just calls the retry wrapper.
- Catch errors per-store so one store failure doesn't block others.
- Return JSON response with results summary: `{ synced: N, failed: N, results: [...] }`. Include `attempts` count from each result.
- Import `server-only` is NOT needed here (API route, not a module).
- Set `export const dynamic = 'force-dynamic'` and `export const maxDuration = 300` (5 minutes — needed to accommodate retry backoff delays of up to ~21 minutes total across all attempts. On Vercel Hobby plan this may need to be reduced; the retry delays can be lowered for Hobby tier if needed).

**src/actions/xero/triggerManualSync.ts** — Server Action for manual sync:
- `'use server'` + `import 'server-only'`
- Verify owner auth: `createSupabaseServerClient()`, `getUser()`, extract `store_id` from `app_metadata`.
- If no user or no store_id: return `{ success: false, message: 'Not authenticated' }`.
- Call `executeManualSync(storeId)` (no retry — owner can manually retry from UI).
- Call `revalidatePath('/admin/integrations')` to refresh sync log.
- Return the result from executeManualSync: `{ success, message, invoiceNumber }`.
