## Spacing Scale

Declared values (multiples of 4, enforced via CSS custom properties in globals.css):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding within badges |
| sm | 8px | Compact element spacing, form label-to-input gap |
| md | 16px | Default element spacing, card internal padding |
| lg | 24px | Section padding, form section spacing |
| xl | 32px | Layout gaps between page sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing (not expected in this phase) |

Exceptions:
- Sync log row min-height: 48px (readable data row with multiple columns)
- Account code form input height: 48px (matches sync log row height and banner height, standard set value)
- Banner alert: full-width strip at top of admin content area, 48px min-height to ensure prominence

## Typography

All roles use DM Sans (font-sans) except page heading which uses Satoshi (font-display).

Scale: 4 sizes maximum, 2 weights maximum.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Page heading | 24px (2xl) | 600 (Satoshi) | 1.2 | "Integrations" page title |
| Section heading | 18px (lg) | 600 (DM Sans) | 1.3 | "Xero Connection", "Account Codes", "Sync Log" section headers |
| Body | 16px (base) | 400 (DM Sans) | 1.5 | Descriptive copy, empty state body, form labels |
| Label / Caption / Data | 14px (sm) | 400 or 600 (DM Sans) | 1.4 | See note below |

**14px usage note:** Two visual variants share the 14px size — weight distinguishes them:
- 14px weight 600: table column headers, form field labels, status badge text, CTAs at small size
- 14px weight 400 + muted color (--color-muted): captions, timestamps, error messages below form fields, sync log secondary values

Tabular data (sync log amounts, invoice numbers): use `font-feature-settings: 'tnum' 1` on 14px DM Sans — consistent with existing admin tables (DESIGN.md data/tables rule).

## Color

All values from established globals.css @theme block.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #FAFAF9 (--color-bg) | Main page background |
| Secondary (30%) | #F5F5F4 (--color-surface) / #FFFFFF (--color-card) | Page content area; white for cards (connection card, account code form, sync log rows) |
| Accent (10%) | #E67E22 (--color-amber) | Reserved — see list below |
| Destructive | #DC2626 (--color-error) | Disconnect Xero button only |

**Accent (#E67E22) reserved for:**
1. "Connect to Xero" primary CTA button (connected state: "Reconnect to Xero")
2. "Sync Now" / "Run Manual Sync" button
3. Active sidebar nav link indicator (left border, consistent with existing pattern)

**Semantic colors:**
- Success (#059669): sync log "Success" status badge background/text
- Error (#DC2626): sync log "Failed" status badge; banner alert background tint; error messages
- Warning (#D97706): sync log "Pending / Retrying" status badge; Xero disconnection banner alert
- Info (#3B82F6): informational help text (e.g. "Sync runs automatically at 2am NZST")

**Disconnection banner:**
- Background: #FEF3C7 (amber-50 equivalent, warm warning tint)
- Border: 1px solid #D97706 (--color-warning)
- Text: #92400E (amber-800 equivalent)
- Icon: warning triangle inline SVG, 16px

**Disconnect confirmation Keep Connected button:**
- Keep Connected — ghost button, --color-navy text, no fill, no border

## Focal Point

**Screen: /admin/integrations**

- **Disconnected state:** XeroConnectButton is the focal point. It is the primary amber CTA, centered-left in the Xero Connection card, with the most visual weight on the page. All other content (account codes, sync log) is secondary or hidden.
- **Connected state:** XeroSyncButton ("Sync Today's Sales") is the focal point. It is the primary amber CTA in the Sync section. The sync status indicator (last synced timestamp, most recent sync log row) is the secondary focal element directly below it.

## Page Layout: /admin/integrations

Single-page layout within existing admin shell (sidebar 240px + main content area). Content is a vertical stack of three card sections:

```
[Page heading: "Integrations" — 24px Satoshi 600]
[Subheading: "Connect your accounting software to sync sales automatically." — 16px DM Sans muted]

[Card: "Xero Connection"]
  Connection status pill + Connect/Disconnect buttons
  OAuth redirect initiator

[Card: "Account Codes"]  (only visible when connected)
  XeroAccountCodeForm — 3 fields
  Save button (navy primary)

[Card: "Sync Log"]
  XeroSyncButton (amber) — "Sync Today's Sales" + last synced timestamp
  XeroSyncLog table — 30 rows max
```

Card styling: `bg-card border border-[var(--color-border)] rounded-[var(--radius-lg)] p-6 shadow-sm` — consistent with all existing admin cards.

## Interaction States

### Xero Connect Button
| State | Visual |
|-------|--------|
| Disconnected (no connection) | Amber filled button "Connect to Xero" |
| Connected | Green badge "Connected" + ghost "Disconnect" button (red text, no fill) |
| Disconnected (was connected) | Warning amber badge "Disconnected" + amber filled "Reconnect to Xero" |
| OAuth in-progress | Button disabled, "Connecting..." label, no spinner (server redirect) |

### Sync Button
| State | Visual |
|-------|--------|
| Idle | Amber filled "Sync Today's Sales" |
| Loading | Button disabled, inline spinner (16px SVG spin animation, 300ms ease-in-out loop), "Syncing..." label |
| Success | Green inline message "Synced — Invoice #[number]" replaces button feedback area for 5 seconds, then clears |
| Error | Red inline message "[error description]. Check sync log for details." persists until next action |
| Not connected | Button hidden (Account Codes section hidden; Sync Log visible read-only with past data) |

### Sync Log Status Badges
| Status | Background | Text | Border |
|--------|------------|------|--------|
| Success | #D1FAE5 | #065F46 | none |
| Failed | #FEE2E2 | #991B1B | none |
| Pending | #FEF3C7 | #92400E | none |

Badge sizing: `px-2 py-0.5 rounded-full text-xs font-medium` — consistent with OrderStatusBadge.tsx pattern.

### Disconnect Xero (Destructive)
Clicking "Disconnect" shows an inline confirmation within the connection card:

```
[Confirmation box, bordered red, rounded-md, p-4]
"Disconnect Xero? Daily sync will stop and account codes will be cleared.
 You can reconnect at any time."
[Keep Connected — ghost button, --color-navy text, no fill]  [Disconnect — red filled button "Disconnect Xero"]
```

No separate modal. Inline within the card. Pressing Escape or clicking "Keep Connected" dismisses.
