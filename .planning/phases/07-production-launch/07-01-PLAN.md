The script validates production readiness by checking environment variables and configuration:

```typescript
#!/usr/bin/env npx tsx
```

Required env var checks (exit 1 if any fail):
1. `NEXT_PUBLIC_SUPABASE_URL` — must be set and must NOT contain `127.0.0.1` or `localhost`
2. `NEXT_PUBLIC_SUPABASE_ANON_KEY` — must be set and must NOT equal `your-anon-key`
3. `SUPABASE_SERVICE_ROLE_KEY` — must be set and must NOT equal `your-service-role-key`
4. `STAFF_JWT_SECRET` — must be set and must be at least 64 characters (32 bytes hex)
5. `STRIPE_SECRET_KEY` — must be set (warn if starts with `sk_test_` but do not fail — test keys are valid per D-05)
6. `NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY` — must be set
7. `STRIPE_WEBHOOK_SECRET` — must be set and must start with `whsec_`

Output format:
- Each check prints: `[PASS] VAR_NAME` or `[FAIL] VAR_NAME: reason` or `[WARN] VAR_NAME: reason`
- At end, print summary: `N/7 checks passed, M warnings`
- Exit code 0 if all pass (warnings OK), exit code 1 if any fail

Add a `check:env` script to package.json: `"check:env": "tsx scripts/check-production-env.ts"`
