Human performs these steps in order:

**Step 1: Register Stripe webhook endpoint**
1. Go to https://dashboard.stripe.com/test/webhooks (note: /test/ path for test mode per D-05)
2. Click "Add endpoint"
3. Endpoint URL: `https://<your-project>.vercel.app/api/webhooks/stripe` (use the production Vercel URL from Plan 02)
4. Under "Select events to listen to", click "Select events"
5. Search for and select: `checkout.session.completed`
6. Click "Add endpoint"
7. On the endpoint detail page, click "Reveal" under Signing secret
8. Copy the `whsec_...` value

**Step 2: Update STRIPE_WEBHOOK_SECRET in Vercel**
1. Go to Vercel Dashboard > Project > Settings > Environment Variables
2. Find `STRIPE_WEBHOOK_SECRET` (currently set to placeholder)
3. Update value to the `whsec_...` signing secret from Step 1
4. Save
5. Trigger a redeploy: Vercel Dashboard > Deployments > most recent > "..." > Redeploy (or push an empty commit)
6. Wait for the redeploy to complete

**Step 3: Test the full checkout flow**
1. Visit the production storefront: `https://<your-project>.vercel.app`
2. If no products exist yet (before CSV import), you need at least 1 product. Create one manually:
   - Go to Admin > Products > Add Product
   - Name: "Test Product", Price: $10.00, Stock: 100, Category: (create one if needed)
   - Save
3. On the storefront, add the product to cart
4. Proceed to checkout (should redirect to Stripe hosted checkout page)
5. Use test card: `4242 4242 4242 4242`, any future expiry, any CVC, any postcode
6. Complete payment
7. Verify you are redirected back to a success/order confirmation page
8. Verify in Admin > Orders that the order appears
9. Verify in Stripe Dashboard > Developers > Webhooks > endpoint > "Recent deliveries" that the webhook was delivered with status 200

**Troubleshooting if webhook fails (status != 200):**
- Check Stripe webhook delivery response body for error details
- Most common: `STRIPE_WEBHOOK_SECRET` mismatch — re-copy the signing secret
- Second most common: the endpoint URL is wrong — ensure it matches exactly: `/api/webhooks/stripe`
- After fixing, you can click "Resend" on the failed delivery in Stripe Dashboard

Human performs these steps (per D-01, D-02, D-03):

**Step 1: Prepare the CSV file**
Create a CSV file with 200+ product rows. Required columns:
- `name` (required) — product name
- `price` (required) — price in dollars, e.g., `12.50` (the import converts to cents)

Optional columns (strongly recommended):
- `barcode` — EAN-13 or UPC-A barcode number
- `category` — category name (creates category if it does not exist)
- `stock_level` — integer stock quantity

Example CSV header and rows:
```csv
name,price,barcode,category,stock_level
Widget A,12.50,9400000000001,Widgets,100
Gadget B,24.99,9400000000002,Gadgets,50
```

Per D-02: This data is created from scratch by the owner — no migration from an existing system.

**Step 2: Import via admin dashboard**
1. Log in to admin at `https://<your-project>.vercel.app/admin/login`
2. Navigate to Products > Import CSV (or look for "Import Products" button)
3. Upload the CSV file
4. Map columns: the column mapper UI lets you match CSV headers to product fields
5. Preview the import — check row count and any validation errors
6. Click "Import [N] Products" to execute
7. Verify the success summary: "Import complete. [N] products added to your catalog."
8. If any rows failed, review errors, fix CSV, and re-import the failed rows

**Step 3: Verify product count**
In Admin > Products, verify the product list shows 200+ products. Alternatively, run in Supabase SQL Editor:
```sql
SELECT COUNT(*) FROM public.products WHERE store_id = '<your-store-id>';
```
Should return 200 or more.

**Step 4: Upload product images (ongoing)**
Per D-03: Images are uploaded individually through the admin product editor.
1. Admin > Products > click a product
2. In the product editor drawer, use the image upload area (drag-drop or click)
3. Upload a photo for each product
4. This is an ongoing task — not all 200+ images need to be uploaded immediately
5. At minimum, upload images for the most visible/popular products before go-live

**Step 5: Verify storefront displays products**
1. Visit the production storefront: `https://<your-project>.vercel.app`
2. Verify the product grid shows products from the catalog
3. Verify categories are visible and filterable
4. Verify product images display for items that have them
5. Verify prices display correctly in NZD format ($XX.XX)
