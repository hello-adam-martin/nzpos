## Copywriting Contract

Phase 7 has no new UI screens. The copywriting contract covers the two main admin flows that will be used intensively during this phase: CSV import and product image upload.

| Element | Copy |
|---------|------|
| Primary CTA — CSV import start | "Import Products" |
| Primary CTA — CSV upload step | "Upload CSV" |
| Primary CTA — confirm import | "Import [N] Products" |
| Primary CTA — product image upload | "Upload Photo" |
| Empty state — products list (before import) | "No products yet" / "Upload a CSV to add your first products, or create them one by one." |
| Empty state — orders list | "No orders yet" / "Orders will appear here once your store goes live." |
| Error state — CSV parse error | "We couldn't read this file. Check that it's a CSV with the required columns (name, price) and try again." |
| Error state — row validation failure | "X rows couldn't be imported. Review the errors below, fix your CSV, and re-upload." |
| Error state — image upload failure | "Image upload failed. Check your connection and try again. Your product data was saved." |
| Error state — Stripe checkout failure (storefront) | "Payment was not completed. Your card was not charged. Try again or contact us." |
| Destructive — delete product | "Delete Product": "This will permanently remove the product and cannot be undone. Any orders containing this product will not be affected." |
| Destructive — clear import | "Cancel Import": "Imported rows will not be saved. Return to the upload step?" |
| Confirmation label — import complete | "Import complete. [N] products added to your catalog." |
| Smoke test label — Stripe test mode banner | "Test mode active. Payments are simulated. No real charges will occur." |

**NZ-specific copy notes:**
- Use "GST" not "Tax" throughout.
- Use "EFTPOS" not "debit card".
- Currency format: NZD with dollar sign prefix, no space — "$12.50".
- Date format: DD/MM/YYYY (NZ standard).

## Phase-Specific UI Notes

### Admin CSV Import Flow

The existing `CSVImportFlow.tsx` (4-step: upload, map columns, preview, import) is the primary UI for loading 200+ products. No changes needed. Key interaction contracts already implemented:

- Step indicator: shows current step, completed steps in navy, upcoming in muted
- Column mapper: dropdowns for mapping CSV headers to product fields
- Import preview table: DM Sans tabular-nums, alternating surface rows
- Import summary bar: success count in success green, error count in error red
- Import button: amber accent, disabled until column mapping is valid

### Product Image Upload

Images uploaded individually via the existing product editor drawer (`ProductFormDrawer.tsx`). The `ProductImagePicker.tsx` component handles drag-drop or click-to-select. No changes needed.

### Stripe Test Mode Indicator

During the Stripe test-key validation period (per D-05), a banner should be visible in the admin dashboard and storefront to prevent the owner from treating test checkouts as real sales. If not already implemented, implement as:

- Location: fixed top of admin layout, below nav bar
- Background: `--color-warning` (#D97706) at 10% opacity, `--color-warning` border, `--color-warning` text
- Copy: "Test mode active. Payments are simulated. No real charges will occur."
- Dismissal: none — persistent while `STRIPE_SECRET_KEY` starts with `sk_test_`
- Detection: read `process.env.STRIPE_SECRET_KEY?.startsWith('sk_test_')` in a Server Component

### Smoke Test Checklist States

The validation sequence (DEPLOY-01 through DEPLOY-04) produces visible feedback through existing UI only:

| Smoke test | UI surface | Expected visual state |
|-----------|-----------|----------------------|
| Admin login works | Admin login page → dashboard | Dashboard hero card shows today's stats |
| Products load | Admin → Products | Product data table shows rows |
| Storefront renders | `/` (public) | Product grid visible with images |
| Stripe test checkout | Storefront → checkout | Stripe hosted page loads, test card succeeds |
| CSV import | Admin → Products → Import CSV | 4-step flow completes, success summary shown |
