## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| 2xs | 2px | Border offsets, hairline gaps |
| xs | 4px | Icon gaps, badge padding, inline padding |
| sm | 8px | Compact element spacing, button icon gap |
| md | 16px | Default form field spacing, card inner padding |
| lg | 24px | Section padding, card outer margin |
| xl | 32px | Layout column gap, page-level padding |
| 2xl | 48px | Major section breaks, hero card grouping |
| 3xl | 64px | Page-level top padding |

Exceptions:
- Dashboard hero stat cards: minimum height 96px for scannable at-a-glance reading
- Data table rows: minimum 48px height (matches established product table pattern)
- Cash denomination input rows: 44px height for comfortable finger targeting on shared desktop/tablet
- Modal/drawer overlay: full-height slide-in panel, 480px wide on desktop

Source: DESIGN.md spacing scale, confirmed in existing `ProductDataTable.tsx` row heights

## Color

| Role | Value | CSS Token | Usage |
|------|-------|-----------|-------|
| Dominant (60%) | #FAFAF9 | `--color-bg` | Main page background, admin layout body |
| Secondary (30%) | #F5F5F4 / #1E293B | `--color-surface` / `--color-navy` | Cards (#F5F5F4), sidebar nav (#1E293B), table header (#1E293B) |
| Accent (10%) | #E67E22 | `--color-amber` | See reserved list below |
| Destructive | #DC2626 | `--color-error` | Refund button, refund confirmation dialog, error alerts |

**Accent (#E67E22) reserved for:**
1. "Process Refund" confirmation button (final step only — not the initial refund trigger button, which is navy)
2. Low stock alert badge border/icon tint on dashboard
3. Active nav item left border in sidebar (already established in `AdminSidebar.tsx`)
4. "Export CSV" primary action button on report pages
5. Cash-up "Close Session" action button

**Accent is NOT used for:**
- Informational stat cards on dashboard (use navy)
- Filter bar controls (use border/surface)
- Table sort icons (use navy/text-muted)

**Semantic colors (used exactly as named):**
- `--color-success` (#059669): Order status COMPLETED badge, COLLECTED status, positive variance (cash over)
- `--color-warning` (#D97706): Low stock alert pill, PENDING_PICKUP status, negative variance (cash short) when < $5
- `--color-error` (#DC2626): REFUNDED status badge, cash variance > $5 short, form errors
- `--color-info` (#3B82F6): Order status READY badge, report info callouts

Source: DESIGN.md color palette, confirmed in `globals.css` @theme, consistent with `ProductStatusBadge.tsx` badge pattern

## Component Inventory

New components required for this phase (follow existing patterns):

### Dashboard Page
- `DashboardHeroCard` — stat card with large Satoshi display number + DM Sans label. White card bg, border, sm shadow. Props: label, value, subLabel (e.g. "POS: 12 / Online: 8").
- `LowStockAlertList` — unordered list below hero cards. Each row: product name (semibold), current stock (mono, warning color if ≤ threshold), threshold (text-muted). Max 10 items; "View all" link when more.
- `ChannelBadge` — pill badge: "POS" (navy fill) or "Online" (info fill). Reuse pattern from `ProductStatusBadge`.

### Order List Page (`/admin/orders`)
- `OrderDataTable` — follows `ProductDataTable` pattern exactly. Navy header row, 48px min row height, border-border, sortable columns, click row to open drawer.
- `OrderFilterBar` — follows `ProductFilterBar` pattern. Filters: status, channel, payment method, date range. Search input for order ID.
- `OrderDetailDrawer` — slide-in right panel (480px), follows `ProductFormDrawer` animation (250ms ease-out). Sections: header (order ID + date), line items table with GST, payment info block, status history, action buttons.
- `OrderStatusBadge` — maps status → semantic color. COMPLETED=success, REFUNDED=error, PENDING_PICKUP=warning, READY=info, COLLECTED=success (lighter variant).

### Refund Flow (within OrderDetailDrawer)
- "Refund Order" button: navy bg, white text, full width, positioned at bottom of drawer. NOT amber — this is a serious action, not a primary happy-path action.
- `RefundConfirmationStep` — replaces drawer content. Shows order total being refunded, `RefundReasonSelect` dropdown, "Restock items?" toggle, amber "Confirm Refund" button + ghost "Cancel" button.
- `RefundReasonSelect` — controlled select input. Options: "Customer request", "Damaged", "Wrong item", "Other". Required before Confirm is enabled.

### Cash-Up
- `CashSessionBanner` — shown in POS top bar when session is open. Displays elapsed time. Tapping opens `CashUpModal`.
- `CashUpModal` — full-screen modal (not drawer). Two states: Open Session (enter opening float) and Close Session (enter counted cash + optional denomination breakdown).
- `DenominationBreakdown` — optional collapsible section within Close Session. Grid of NZ note/coin inputs that sum to total. Notes: $100, $50, $20, $10, $5. Coins: $2, $1, $0.50, $0.20, $0.10.
- `CashVarianceSummary` — read-only block after cash counted: Expected Cash, Actual Cash, Variance. Variance styled: positive=success, within $2=text-muted, >$2 short=warning, >$5 short=error.

### Reports Page (`/admin/reports`)
- `ReportDateRangePicker` — tab-style preset selector (Today / Yesterday / This Week / This Month / Last Month / GST 1-month / GST 2-month / GST 6-month) + Custom Range date inputs. Selected preset: navy fill pill; others: ghost pills with border.
- `SalesSummaryTable` — date-grouped rows with daily totals. Mono font for all numeric columns.
- `SalesBarChart` — simple vertical bar chart using Recharts (see chart library decision below). Navy bars, amber accent for the selected/hovered bar. No decorative grid lines — only bottom axis and left axis.
- `ChannelBreakdownChart` — donut/pie chart. Navy segment = POS, info-blue segment = Online.
- `GSTSummaryBlock` — two-mode view toggled by tabs: "Summary" (total sales, GST collected, GST-exclusive total) and "Detail" (per-line breakdown table with order ID, line amount, GST amount). Mono font throughout.
- `ExportCSVButton` — amber bg, white text, icon (download SVG), appears top-right of each report section.

### Chart Library Decision (Claude's Discretion)
Use **Recharts** (recharts@2.x). Rationale: lightweight, React-native, composable primitives that match the project's utility-first approach. Supports `ResponsiveContainer` for fluid widths. No build-step complexity. Well-maintained. Alternative considered: Victory (heavier, more opinionated styling). Tremor was rejected — it is a component library that conflicts with the custom design system.

## Interaction Contracts

### Drawer Behavior (OrderDetailDrawer)
- Opens from right edge. Width: 480px on desktop, full-width on screens < 640px.
- Overlay: `rgba(0,0,0,0.4)` behind drawer.
- Animation: slide-in 250ms ease-out (enter), slide-out 200ms ease-in (exit). Matches `ProductFormDrawer` existing pattern.
- Clicking overlay closes drawer (unless refund confirmation step is active — requires explicit cancel).
- Escape key closes drawer.

### Refund Confirmation Flow
1. Owner clicks "Refund Order" (navy button, bottom of drawer).
2. Drawer content replaced by `RefundConfirmationStep` (no new modal — in-place replacement within drawer).
3. Owner selects refund reason (required), toggles restock, clicks amber "Confirm Refund".
4. Loading state on button during Server Action. On success: drawer shows read-only refunded state with green badge. On error: inline error message above buttons.
5. Cancel returns to order detail view.

### Cash-Up Flow
1. POS top bar shows "Open Session" button (ghost, small) when no session is active.
2. When session is open: `CashSessionBanner` shows "Session open · 2h 34m" with a "Close" button.
3. Close Session triggers `CashUpModal`:
   - Step 1: Enter total counted cash (required). Optional: expand denomination breakdown.
   - Step 2: Review variance summary. "Confirm Close" (amber) or "Back" (ghost).
4. Admin dashboard Cash-Up section shows last 5 sessions with open time, close time, variance.

### Low Stock Alerts
- Dashboard list refreshes on page load (Server Component revalidation).
- No real-time updates — consistent with refresh-on-transaction decision from STATE.md.
- Products at exactly the reorder threshold show warning color. Products at 0 show error color.

### Report Date Range
- Default on page load: "Today".
- Changing preset immediately fetches new data (Server Action or route navigation with search params).
- Custom Range: two date inputs (from/to), "Apply" button activates fetch.
- Active preset pill: navy fill, white text, no border.

### Order Table Pagination
- Server-side pagination. 50 rows per page (default).
- Pagination controls: Previous / Next with current page indicator. Positioned below table.
- URL search params encode current page, filters, and sort state (bookmarkable).

## Phase-Specific Notes for Executor

1. **Follow `ProductDataTable` and `ProductFormDrawer` exactly** for `OrderDataTable` and `OrderDetailDrawer`. Same class names, same interaction patterns, same spacing. Do not invent new table or drawer patterns.

2. **Admin sidebar extension.** Add to `AdminSidebar.tsx` navLinks array: `{ href: '/admin/orders', label: 'Orders' }`, `{ href: '/admin/reports', label: 'Reports' }`, `{ href: '/admin/cash-up', label: 'Cash-Up' }`. Maintain existing active state styling (amber left border, `bg-white/10`).

3. **formatNZD is mandatory** for all monetary display. Never format cents manually. Import from `src/lib/money.ts`.

4. **GST calculations in reports must use `src/lib/gst.ts`** (gstFromInclusiveCents, calcOrderGST). No inline GST math.

5. **Integer cents only** — all database reads return cents. Divide by 100 only in `formatNZD`. Never store or pass floats.

6. **Recharts usage.** Import only `BarChart`, `Bar`, `PieChart`, `Pie`, `Cell`, `XAxis`, `YAxis`, `Tooltip`, `ResponsiveContainer`. Do not import the full library. Use navy (#1E293B) as primary bar fill, amber (#E67E22) for active/hovered state. Remove default Recharts borders and background.

7. **Cash denomination inputs** — each is a number input with min=0. Auto-calculate total from denominations client-side using `useReducer`. The total field is the source of truth for Server Action submission — denominations are cosmetic/UX only.

8. **Order status badges** must use semantic CSS tokens, not hardcoded hex values: `bg-success`, `bg-warning`, `bg-error`, `bg-info` at 15% opacity backgrounds with full-opacity text variants.
