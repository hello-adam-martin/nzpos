    - Same card shell, icon uses `--color-text-light` instead of navy

    ```typescript
    'use client'
    ```
    - Receives `storePlans` (has_xero, has_email_notifications, has_custom_domain, stripe_subscription_id etc), `stripeCustomerId`, `addons` metadata as props

    - Renders AddOnCard for each add-on with computed status (active/trial/inactive based on store_plans flags + subscription data)
    - Renders "Billing management" section with portal button:

      - If no stripeCustomerId: text "Subscribe to an add-on to access the billing portal." (per UI-SPEC Phase-Specific Note 4)

    Portal button style: `bg-[var(--color-navy)] text-white text-sm font-semibold px-4 py-2 rounded-[var(--radius-md)] hover:bg-[var(--color-navy-light)]`
