    Per D-09: ALL Xero Server Actions and email notification Server Actions must call requireFeature() before executing. Per D-07: use `requireDbCheck: true` for mutations (Xero sync, email send) to catch cancellations between JWT refreshes.

    **src/actions/xero/triggerManualSync.ts** — Add at the top of the function body, before any existing logic:
    ```typescript
    import { requireFeature } from '@/lib/requireFeature'

    // Add as first line inside the function:
    const gate = await requireFeature('xero', { requireDbCheck: true })
    if (!gate.authorized) {
      return { success: false, message: 'Xero subscription required' }
    }
    ```

    **src/actions/xero/saveXeroSettings.ts** — Same pattern. Add requireFeature('xero', { requireDbCheck: true }) as first operation. Return early with `{ error: 'Xero subscription required' }` or match its existing return shape on failure.

    **src/actions/xero/disconnectXero.ts** — Same pattern. Add requireFeature('xero', { requireDbCheck: true }) as first operation. Return `{ success: false, error: 'Xero subscription required' }` matching existing return type.

    **src/lib/email.ts** — The `sendEmail` function is a utility, not a Server Action. Add the gate inside:
    ```typescript
    import { requireFeature } from '@/lib/requireFeature'

    // Inside sendEmail, before the Resend call:
    const gate = await requireFeature('email_notifications', { requireDbCheck: true })
    if (!gate.authorized) {
      console.log('[email] Email notifications not active — skipping send')
      return  // Silent skip — callers don't need to handle this
    }
    ```

    IMPORTANT: `sendEmail` currently has `import type React from 'react'` and uses Resend. The requireFeature check should be added BEFORE the `getResend()` call. If the function signature returns void, the early return is sufficient. If it returns a result, return accordingly.

    Note: email.ts already has `import 'server-only'` — no conflict with requireFeature which also uses server-only.
