  <action>
    **Install knip (per D-06):**

    1. Run: `npm install -D knip`

    **Create knip.json configuration:**

    2. Create `knip.json` in the project root with this configuration:
       ```json
       {
         "$schema": "https://unpkg.com/knip@6/schema.json",
         "entry": [
           "src/app/**/{page,layout,route,loading,error,not-found,default}.{ts,tsx}",
           "src/app/**/actions.ts",
           "src/actions/**/*.ts",
           "src/middleware.ts",
           "src/lib/supabase/*.ts",
           "src/test/setup.ts",
           "src/**/*.test.{ts,tsx}",
           "vitest.config.mts",
           "next.config.ts",
           "scripts/**/*.ts",
           "postcss.config.mjs"
         ],
         "project": [
           "src/**/*.{ts,tsx}",
           "scripts/**/*.ts"
         ],
         "ignore": [
           "src/types/**",
           "src/emails/**",
           ".planning/**",
           "tests/e2e/**"
         ],
         "ignoreDependencies": [
           "server-only",
           "@types/*",
           "sharp"
         ]
       }
       ```

    Key configuration decisions:
    - Entry points include ALL Next.js App Router file conventions (page, layout, route, loading, error, not-found, default)
    - `src/actions/**/*.ts` as entry points because Server Actions are imported by page components
    - `src/**/*.test.{ts,tsx}` as entries so test-only exports are not flagged as unused
    - `server-only` ignored because it's a side-effect import with no exports
    - `@types/*` ignored because they're type-only dev dependencies
    - `sharp` ignored because Next.js auto-detects it at build time (not imported in source)
    - `src/emails/**` ignored because React Email templates are rendered by the email library, not imported by app code

    **Run knip and analyze output:**

    3. Run: `npx knip` and capture output
    4. Review all flagged items. For each category:

    **Unused files (per D-07):** Delete them. Git history preserves anything needed later. Before deleting, verify the file is truly unused (not dynamically imported or referenced via string).

    **Unused exports:** Remove the `export` keyword if the function/type is still used internally. Delete entirely if the function/type is not used anywhere.

    **Unused dependencies (per D-08):** Run `npm uninstall {package}` for each unused dependency. Verify the build still passes after removal.

    **Unlisted dependencies:** If knip reports dependencies used in code but not in package.json, add them.

    5. After all removals, run `npx knip` again to verify clean output.
    6. Run `npm run build` to verify the build still succeeds.
    7. Run `npm run test` to verify all tests still pass.

    **Important:** Do NOT remove anything in `src/types/`, `src/emails/`, or `.planning/` — these are excluded from knip analysis. Do NOT remove `server-only` from dependencies. Do NOT remove type-only `@types/*` packages that TypeScript needs.
  </action>
