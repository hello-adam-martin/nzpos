## Spacing Scale

All spacing uses CSS custom properties — never hardcoded px values. Defined below as reference.

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| `var(--space-xs)` | 4px | Icon-to-text gap in compact feature cards; checklist item gaps |
| `var(--space-sm)` | 8px | Inline element gaps (nav links, checklist icons); sub-copy margin below CTA button |
| `var(--space-md)` | 16px | Default content gap; horizontal padding (mobile); feature card title top margin |
| `var(--space-lg)` | 24px | Section horizontal padding (desktop `md:`); between feature group heading and cards; add-on card grid gap |
| `var(--space-xl)` | 32px | Layout column gap; hero padding (mobile); free tier card padding; CTA button padding-x |
| `var(--space-2xl)` | 48px | Hero section vertical padding (mobile) |
| `var(--space-3xl)` | 64px | Features, Pricing, CTA section vertical padding |

Exceptions: none — no POS touch targets in this phase; all interactive elements are standard nav links and one CTA button.

Source: DESIGN.md spacing scale; confirmed in all Landing*.tsx components.

## Layout Patterns

### Pattern A — Grouped Features (LandingFeatures rewrite)

Section background: `var(--color-bg)`. Full-width section, no max-width on section element.

Inner container: `max-w-[1200px] mx-auto px-[var(--space-md)] md:px-[var(--space-lg)] py-[var(--space-3xl)]`

Section heading: `font-sans text-xl font-bold text-[var(--color-text)] text-center leading-[1.3]`

Feature groups: 3–4 groups, each rendered as a block with a category label above a compact card grid.

Category label: `font-sans text-sm font-bold text-[var(--color-text-muted)] uppercase tracking-wide mb-[var(--space-md)]`

Compact card grid (per group): `grid grid-cols-1 sm:grid-cols-2 md:grid-cols-3 gap-[var(--space-md)]`

Each compact card: `flex gap-[var(--space-sm)] items-start` — no border, no background, no shadow. Icon + text inline.

Icon wrapper: `text-[var(--color-navy)] shrink-0`

Card title: `font-sans text-sm font-bold text-[var(--color-text)]`

Card description: `font-sans text-sm text-[var(--color-text-muted)]`

Between groups: `mt-[var(--space-2xl)]`

**Why compact cards not rich cards:** At 12+ features, bordered rich cards (white bg + shadow + xl padding) create visual overload. Compact inline cards maintain scanability. Rich card style is reserved for pricing only.

### Pattern B — NZ Callout Strip (new LandingNZCallout)

Section background: `bg-[var(--color-navy)]` — full viewport width, no max-width on section element.

Inner container: `max-w-[1200px] mx-auto px-[var(--space-md)] md:px-[var(--space-lg)] py-[var(--space-2xl)]`

Layout: `flex flex-col md:flex-row items-center justify-center gap-[var(--space-xl)] text-center`

Each callout item: `flex flex-col items-center gap-[var(--space-sm)]`

Item icon: 24x24 inline SVG, `text-[var(--color-amber)]` (amber on navy — high contrast, uses accent intentionally for trust badges)

Item label: `font-sans text-base font-bold text-white`

Item sub-label (optional): `font-sans text-sm text-white/70`

Three items:
1. GST-Compliant — "15% GST calculated per line, IRD-compliant"
2. NZD Pricing — "All prices in New Zealand dollars, GST included"
3. Built in NZ — "Made for Kiwi retailers, by Kiwi developers"

**Note:** The NZ callout is the only place amber appears as an icon color on a dark background. This is intentional — amber on navy passes WCAG AA contrast and signals positive attributes without being a CTA.

### Pattern C — Pricing Layout fix (LandingPricing rewrite)

Section background: `bg-[var(--color-surface)]`. Identical structure to current component.

Free tier card: unchanged structure. Updated feature list per D-05 (see Copywriting below).

Add-on grid: change from `grid-cols-1 md:grid-cols-2 max-w-lg mx-auto` to `grid grid-cols-1 md:grid-cols-3 gap-[var(--space-lg)] mt-[var(--space-lg)]` — no max-width, three equal columns.

Each add-on card: unchanged style (`bg-white border border-[var(--color-border)] rounded-lg p-[var(--space-xl)]`). No CTA buttons per D-06.

### Pattern D — SVG Icon Convention

All inline SVGs must exactly match this template:

```tsx
<svg
  xmlns="http://www.w3.org/2000/svg"
  width="24"
  height="24"
  viewBox="0 0 24 24"
  fill="none"
  stroke="currentColor"
  strokeWidth="2"
  strokeLinecap="round"
  strokeLinejoin="round"
  aria-hidden="true"
>
  {/* paths */}
</svg>
```

Color is inherited via `currentColor`. Parent sets `text-[var(--color-navy)]` on light bg or `text-[var(--color-amber)]` on navy bg. Exception: checkmark SVGs in free tier checklist use `className="text-[var(--color-success)] shrink-0"` and `strokeWidth="2.5"`, width/height 16.

## Responsive Contract

All components are mobile-first. The single breakpoint is `md:` (768px).

| Component | Mobile | Desktop (md:) |
|-----------|--------|---------------|
| LandingNav | Wordmark + hamburger (details/summary) | Wordmark + anchor links + auth buttons |
| LandingHero | Single column, copy only | 2-column grid, copy left + iPad mockup right |
| LandingFeatures | 1-column compact cards | 3-column compact card grid per group |
| LandingNZCallout | Vertical stack (flex-col) | Horizontal row (flex-row) |
| LandingPricing | 1-column free card + 1-col add-ons | Free card full-width + 3-col add-on grid |
| LandingCTA | Centered, full-width | Same — no change |

No horizontal scroll permitted at any viewport width. Verify in dev tools at 375px (iPhone SE) and 768px (tablet).
