## Spacing Scale

Declared values from `globals.css` `--space-*` custom properties — all multiples of 4:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, tight inline padding |
| sm | 8px | Compact row padding, button icon gap |
| md | 16px | Default cell padding, form field gap |
| lg | 24px | Section padding, modal body padding |
| xl | 32px | Card outer padding, modal max-width buffer |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions:
- Staff table row minimum height: 48px (comfortable click target for inline action buttons)
- Modal action buttons: minimum 44px height (touch-accessible, consistent with admin pattern)
- PIN display in PinDisplayModal: 36px font-size, centered — display-size exception for one-time visibility emphasis

Source: DESIGN.md spacing section + POS-specific touch target rule applied to admin table rows.

- Inline Edit or Reset PIN row actions (these use ghost/text style in navy)

- `navy` #1E293B — Table column headers, nav sidebar background, primary (non-CTA) buttons
- `border` #E7E5E4 — Table dividers, modal outline, form input borders

## Component Inventory

New components for this phase. All follow existing admin patterns — reference `ProductDataTable.tsx`, `ProductFormDrawer.tsx`, and `CashUpModal.tsx` for structural precedents.

### StaffTable

- Layout: Full-width table, no horizontal scroll at 1024px+. Columns: Name | Role | Status | Actions.
- Column widths: Name (auto, min 180px), Role (120px fixed), Status (100px fixed), Actions (160px fixed, right-aligned).
- Row height: 48px minimum.
- Sorting: No column sorting required (staff lists are short — sort by name server-side).
- Row hover: `bg-surface` (#F5F5F4) on hover. Cursor default (rows are not clickable; actions are).
- Actions column: Three ghost text buttons — "Edit", "Reset PIN", "Deactivate". Separated by a `|` divider in `text-muted`.
- "Deactivate" action: `text-error` color to signal destructive intent. Not hidden — visible but requires confirmation modal.
- Inactive rows: Name and role cells rendered at `text-text-light` (#A8A29E) opacity to indicate deactivated state visually.

### StaffStatusBadge

- Active: `bg-success/10 text-success` pill, `border border-success/20`, text "Active", font-size 12px (xs), weight 600.
- Inactive: `bg-surface text-text-muted` pill, `border border-border`, text "Inactive", same sizing.
- Border-radius: `radius-full` (9999px) — pill shape.

### RoleBadge

- Owner: `bg-navy/10 text-navy` pill.
- Manager: `bg-amber/10 text-amber` pill.
- Staff: `bg-surface text-text-muted` pill.
- Same sizing as StaffStatusBadge. Border-radius: `radius-full`.

### AddStaffModal / EditStaffModal

- Structure: Centered modal overlay, `bg-card` panel, `radius-lg` (12px), `shadow-lg`.
- Max width: 480px.
- Header: Modal title (DM Sans 20px 600) + close button (X, inline SVG, `text-text-muted`, `aria-label="Close"`).
- Body padding: 24px (lg).
- Fields:
  - Name: Text input, full width, `border-border`, `radius-md`, 16px padding, DM Sans 14px.
  - Role: Select/dropdown — Owner | Manager | Staff. Same styling as text input.
  - PIN section (AddStaffModal only): Toggle "Auto-generate PIN" (default on) / "Set manual PIN". When manual: 4-digit PIN input with monospaced digits (`font-mono`). Inline validation on blur.
- Footer: Two buttons right-aligned — dismiss button (ghost, navy border) + primary CTA (amber bg "Add Staff Member" / "Save Changes").
- Animation: open 250ms ease-out, close 200ms ease-in. Source: DESIGN.md motion — medium duration for modals.

### PinDisplayModal

- Structure: Same modal shell as above. Max width: 400px. Cannot be dismissed by clicking outside — must click "Done" button.
- Content layout: Centered vertically.
- PIN display: 4-digit PIN in DM Sans 36px weight 600, `font-feature-settings: 'tnum' 1`, `text-navy`, centered. Background: `bg-surface` pill with 16px padding, `radius-md`.
- "Copy" button: Amber CTA below PIN display, full width, 44px height. On copy: button label changes to "Copied!" for 2 seconds, then reverts. No toast — inline feedback.
- Warning text: DM Sans 14px `text-error`, with warning icon (inline SVG triangle). Text: "Store this PIN somewhere safe — it won't be shown again."
- "Done" button: Below warning, navy ghost button, full width. Dismisses modal.
- No close (X) button in header — must use "Done".

### ConfirmRoleChangeModal

- Structure: Same modal shell. Max width: 400px.
- Header: Modal title (DM Sans 20px 600). No close (X) button — user must choose an action.
- Title: "Change role?" (DM Sans 20px 600).
- Body: "Change [Name] from [Current Role] to [New Role]? They'll be logged out and need to re-enter their PIN." (DM Sans 14px, `text-text`).
- Roles in body text use RoleBadge inline components.
- Footer: "Keep Current Role" (ghost, navy border) + "Change Role" (navy primary, NOT amber — this is a neutral action, not a purchase/add action).
- No destructive styling — role change is a management action, not destructive.

### ConfirmDeactivateModal

- Structure: Same modal shell. Max width: 400px.
- Header: Modal title (DM Sans 20px 600). No close (X) button — user must choose an action.
- Title: "Deactivate [Name]?" (DM Sans 20px 600).
- Body: "They'll be immediately logged out and won't be able to use the POS." (DM Sans 14px `text-text`).
- Warning block: `bg-error/10 border border-error/20 rounded-md p-3` — with warning icon + "This action can be reversed by reactivating them from the staff list." (DM Sans 13px `text-error`).
- Footer: "Keep Active" (ghost, navy border) + "Deactivate" button — `bg-error text-white` (destructive red, NOT amber).
- The "Deactivate" button is the only red primary button used in this phase.

### Loading Skeleton (StaffTable)

- Show 5 skeleton rows at page load. Each row matches 48px height.
- Skeleton cells: Animated pulse `bg-border` bars. Name column: 60% width bar. Role: 80px bar. Status: 60px bar. Actions: hidden (no skeleton for action buttons).
- CSS: `animate-pulse` Tailwind utility, `bg-border` (#E7E5E4) fill.

### AdminSidebar (Manager View)

- When `role === 'manager'`, render only: Dashboard, Orders, Reports, Cash-Up.
- All other links are removed from the DOM — not hidden with CSS visibility/display:none (to prevent accidental disclosure).
- Visual appearance of rendered links: identical to owner view (same active/hover states, same navy sidebar).
- No visual indicator that links are "missing" — the filtered sidebar IS the manager's complete nav.

## Interaction Contracts

### Staff Table — Inline Actions

| Action | Trigger | Result |
|--------|---------|--------|
| Edit | Click "Edit" button | Opens EditStaffModal pre-populated with current name and role |
| Reset PIN | Click "Reset PIN" button | Calls resetStaffPin action immediately (no confirm modal), then opens PinDisplayModal with new PIN |
| Deactivate | Click "Deactivate" button | Opens ConfirmDeactivateModal |

### Role Change Flow

1. Owner opens EditStaffModal, changes role dropdown value.
2. Clicks "Save Changes".
3. If role has changed: ConfirmRoleChangeModal opens (EditStaffModal stays mounted behind overlay).
4. Owner clicks "Change Role" → Server Action fires → modal closes → table revalidates → success toast.
5. If owner clicks "Keep Current Role" → ConfirmRoleChangeModal closes → EditStaffModal remains open with the changed value still selected (no implicit revert — owner must manually revert if they changed their mind).

### PIN Copy Button

- On click: write PIN to clipboard via `navigator.clipboard.writeText()`.
- If clipboard API unavailable (non-HTTPS dev context): show fallback text "Select and copy the PIN above manually."
- Button label change: "Copy" → "Copied!" for 2000ms → "Copy".

### Manager Redirect (D-03)

- Silent. No toast, no error message, no page flash.
- Manager navigating to `/admin/products` → server-side middleware redirect to `/admin/dashboard`.
- Browser URL updates to `/admin/dashboard`. No history entry for the blocked route.

| ConfirmDeactivateModal | idle / submitting |
