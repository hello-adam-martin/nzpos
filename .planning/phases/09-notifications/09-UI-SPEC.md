## Spacing Scale

Declared values from `globals.css` — multiples of 4 only:

| Token | Value | Usage |
|-------|-------|-------|
| 2xs | 2px | Subtle gaps only (not in this phase) |
| xs | 4px | Icon gaps, badge inner padding, inline spacing |
| sm | 8px | Element spacing within POS top bar, toast internals |
| md | 16px | Default element spacing, email section padding |

## Component Inventory

New components this phase. All use existing token system — no new design primitives needed.

### 1. OrderNotificationBadge (new, POS only)

```
Position: absolute overlay on the Pickups nav link in POSTopBar
Size: 20px diameter circle (minimum)
Background: --color-amber (#E67E22)
Text: white, 11px, weight 700
Border: 2px solid #1E293B (navy, to separate from navy bar)
Position: top-right corner of the "Pickups" nav link text
Visibility: hidden when count = 0, shown when count >= 1
Max display: show "9+" when count > 9
```

### 2. MuteToggleButton (new, POS only)

```
Location: POSTopBar right side, before the staff name separator
Size: 44×44px touch target (icon visually ~20px)
Icon: Volume2 (unmuted) / VolumeX (muted) from lucide-react
Color unmuted: text-white/70, hover text-white
Color muted: text-white/40, hover text-white/60
No label text visible — aria-label="Toggle order sound" required
Transition: 150ms (POS motion rule)
```

### 3. NewOrderToast (new, POS only)

```
Position: bottom-right of viewport, 16px margin from edges
Width: 280px fixed
Background: --color-surface (#F5F5F4)
Border: 1px solid --color-border (#E7E5E4)
Border radius: --radius-md (8px)
Shadow: --shadow-md
Padding: 12px 16px

Layout:
  - Icon row: amber dot (8px) + heading ("New order" or "2 new orders")
  - Second line: order total or "NZ$X.XX + NZ$Y.YY" for multiple
  - Auto-dismiss: 6 seconds
  - No close button (auto-dismiss only, badge is the persistent indicator)

Heading: DM Sans 14px weight 600, --color-text
Body: DM Sans 13px weight 400, --color-text-muted
Amber dot: 8px circle, background --color-amber

Animation:
  - Enter: slide up from bottom-right, ease-out, 150ms
  - Exit: fade out, ease-in, 100ms
```

### 4. EmailHeader (shared React Email sub-component)

```
HTML table-based (React Email convention)
Background: #1E293B (navy)
Padding: 24px horizontally, 20px vertically
Store name text: #FFFFFF, 20px, weight 700
Optional: thin 3px amber (#E67E22) bottom border strip
```

### 5. EmailFooter (shared React Email sub-component)

```
Background: #FAFAF9
Padding: 16px 24px
Text: #78716C, 12px, weight 400
Content: store address + phone + hours (context-dependent per template)
```

### 6. LineItemTable (shared React Email sub-component)

```
No actual HTML table — use React Email Section/Row components
```

## Copywriting Contract

### Application UI (POS sound alert + toast)

| Element | Copy |
|---------|------|
| Toast heading — single order | "New order" |
| Toast heading — multiple orders | "{N} new orders" |
| Toast body — single | "NZ${total} — tap Pickups to view" |
| Toast body — multiple | "Tap Pickups to view all orders" |
| Mute toggle aria-label (unmuted) | "Toggle order sound (currently on)" |
| Mute toggle aria-label (muted) | "Toggle order sound (currently off)" |
| Badge accessible label | "{N} unread orders" (aria-label on badge span) |

### Email Templates

| Template | Subject line | Heading | Primary CTA |
|----------|-------------|---------|-------------|
| Online receipt (NOTIF-01) | "Your receipt from {Store Name}" | "Thanks for your order" | none (receipt only) |
| POS receipt (NOTIF-02) | "Your receipt from {Store Name}" | "Here's your receipt" | none (receipt only) |
| Pickup-ready (NOTIF-03) | "Your order is ready for pickup" | "Your order is ready" | none (no link needed) |
| Daily summary (NOTIF-04+05) | "Daily summary — {date}" | "Yesterday at a glance" | none (report only) |

### Empty States

| Context | Copy |
|---------|------|
| Daily summary — zero sale day | "No sales yesterday. The system is working — here's your low stock summary below." |
| Daily summary — no low stock items | No low stock section is rendered (section is omitted, not shown as "none") |
| Toast — sound muted | No toast shown when muted (silent, badge still updates) |

### Error States

| Context | Problem copy | Resolution path |
|---------|-------------|-----------------|
| Email send failure | Silent — logged to console as `[email] Send failed: {error}`. Not shown to users. | Staff sees no error; Resend errors are non-blocking per D-05. |
| Polling API failure | Silent — polling retries on next 30s interval. No UI error shown. | If polling fails consistently, badge simply doesn't update. |
| Sound file fails to load | Silent — no fallback sound, badge + toast still appear. | `new Audio()` error caught and swallowed; visual alert is the fallback. |

### Destructive Actions

None in this phase. The mute toggle is a preference toggle, not a destructive action — no confirmation required.

## Interaction Contracts

### Mute Toggle (NOTIF-06)

```
State machine: unmuted (default) → muted → unmuted
Persistence: localStorage key "pos_sound_muted" = "true" | "false"
Initialisation: read localStorage on component mount; default unmuted if key absent
Icon swap: Volume2 ↔ VolumeX — instant, 0ms (preference feedback)
Opacity change: 150ms transition (POS motion rule)
Sound on toggle: no — toggling mute does not play/silence an in-progress sound
```

### Badge (NOTIF-06)

```
State: tracks integer count of unread online orders
Source: polling response `orders.length` accumulated since last staff view
Reset trigger: when staff navigates to /pos/pickups, reset unread count to 0
  (set localStorage key "pos_last_viewed_pickups" = serverTime from poll response)
Visibility: badge hidden (display none) when count = 0; shown when count >= 1
Max label: show "9+" when count > 9
```

### Toast (NOTIF-06)

```
Trigger: fires once per poll cycle that returns >= 1 new order
Sound: if unmuted, play /public/sounds/chime.mp3 simultaneously with toast show
Multiple orders: single toast ("N new orders"), single chime (D-18)
Auto-dismiss: 6 seconds, no user action required
No close button — toast is informational, badge is the persistent control
If already showing: replace with new toast (don't stack)
```

### Sound File

```
File: /public/sounds/chime.mp3
Format: MP3 (broadest browser compatibility on iPad Safari)
Playback: new Audio('/sounds/chime.mp3').play() — guarded with typeof window !== 'undefined'
Error handling: .catch(() => {}) — silent failure, visual alert continues
Duration: <= 2 seconds (retail chime, not an alarm)
Volume: default (no programmatic volume adjustment)
```

### Email Trigger Points (backend — specified here for executor clarity)

| Trigger | Location | Behaviour |
|---------|----------|-----------|
| NOTIF-01 | `src/app/api/webhooks/stripe/route.ts` after `complete_online_sale` RPC | `void sendEmail(...)` — no await |
| NOTIF-02 | `POSClientShell.tsx` `onEmailCapture` callback | Call new server action → `void sendEmail(...)` |
| NOTIF-03 | `updateOrderStatus.ts` on status → `ready` | `void sendEmail(...)` — no await |
| NOTIF-04+05 | `src/app/api/cron/daily-summary/route.ts` | Await send (cron has no user to block) |

## Email Template Layout Contracts

All templates share the same outer shell: EmailHeader + content sections + EmailFooter.
Container width: 600px max, centered, white background on stone-white canvas.

### OnlineReceiptEmail / PosReceiptEmail (NOTIF-01, NOTIF-02)

```
Structure:
  1. EmailHeader (navy, store name)
  2. "Thanks for your order" / "Here's your receipt" heading — 16px, weight 600
  3. Order number + date — 13px muted
  4. LineItemTable (all items, quantities, prices, discounts)
  5. <Hr>
  6. GST row + Total row
  7. Payment method row (e.g. "Paid via EFTPOS")
  8. <Hr>
  9. EmailFooter (store address + phone)
```

### PickupReadyEmail (NOTIF-03)

```
Structure:
  1. EmailHeader (navy, store name)
  2. Success status pill — "Ready for pickup" (emerald #059669 background, white text, radius-full)
  3. "Your order is ready" heading — 20px, weight 700
  4. Order summary (condensed — item names + quantities only, no prices)
  5. <Hr>
  6. Collection details section:
     - Store address
     - Phone number
     - Opening hours (from store record)
  7. EmailFooter
```

### DailySummaryEmail (NOTIF-04 + NOTIF-05)

```
Structure:
  1. EmailHeader (navy, store name + "Daily Summary" subtitle)
  2. Date heading: "Yesterday at a glance — {date}"
  3. Hero stats row (3 cells, inline or stacked):
     - Total sales count
     - Total revenue (NZD, excl. GST)
     - GST collected
  4. Revenue by payment method table:
     - EFTPOS | NZ$X.XX
     - Cash   | NZ$X.XX
     - Online | NZ$X.XX
  5. <Hr>
  6. Top 5 products section (ordered by quantity sold):
     - Rank number + product name + quantity + revenue
  7. <Hr>
  8. Low stock section (only rendered if >= 1 product below reorder_threshold):
     - Section heading: "Low stock — action needed"
     - Table: Product name | Current stock | Reorder threshold
  9. Zero-sale message (only if total sales = 0):
     "No sales yesterday. The system is working."
  10. EmailFooter
```
