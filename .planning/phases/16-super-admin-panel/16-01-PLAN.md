    Create all 4 Server Actions following this exact pattern for each:

    **Common pattern (all four actions):**
    ```typescript
    'use server'
    import 'server-only'
    import { z } from 'zod'
    import { createSupabaseServerClient } from '@/lib/supabase/server'
    import { createSupabaseAdminClient } from '@/lib/supabase/admin'
    import { revalidatePath } from 'next/cache'
    ```

    Each action: (1) auth check via `supabase.auth.getUser()` + `app_metadata.is_super_admin === true`, (2) Zod validate with `safeParse`, (3) write via admin client, (4) insert audit log to `super_admin_actions`, (5) `revalidatePath('/super-admin/tenants')`.

    **suspendTenant.ts** (per D-09, D-10, D-11):
    - Schema: `{ storeId: z.string().uuid(), reason: z.string().min(1).max(500) }`
    - Accepts FormData (extract storeId, reason)
    - First query stores to get slug: `admin.from('stores').select('slug').eq('id', storeId).single()`
    - Update: `admin.from('stores').update({ is_active: false, suspended_at: new Date().toISOString(), suspension_reason: reason }).eq('id', storeId)`
    - Call `invalidateCachedStoreId(slug)` from `@/lib/tenantCache` after successful update
    - Audit: `admin.from('super_admin_actions').insert({ super_admin_user_id: user.id, action: 'suspend', store_id: storeId, note: reason })`
    - revalidatePath('/super-admin/tenants') and revalidatePath(`/super-admin/tenants/${storeId}`)

    **unsuspendTenant.ts** (per D-11):
    - Schema: `{ storeId: z.string().uuid(), reason: z.string().max(500).optional() }`
    - Accepts FormData (extract storeId, reason)
    - Update: `admin.from('stores').update({ is_active: true, suspended_at: null, suspension_reason: null }).eq('id', storeId)`
    - Audit: insert with action='unsuspend', note = reason or 'No reason given'
    - revalidatePath both paths

    **activateAddon.ts** (per D-13, D-14):
    - Schema: `{ storeId: z.string().uuid(), feature: z.enum(['xero', 'email_notifications', 'custom_domain']) }`
    - Accepts FormData (extract storeId, feature)
    - Use `FEATURE_TO_COLUMN` from `@/config/addons` to get the column name (e.g., `has_xero`)
    - Derive override column: `${column}_manual_override` (e.g., `has_xero_manual_override`)
    - Guard: query `store_plans` first. If column is true AND override is false, return `{ error: 'Already active via Stripe subscription' }` (prevents setting manual override on Stripe-paid addon)
    - Write: `admin.from('store_plans').update({ [column]: true, [overrideColumn]: true }).eq('store_id', storeId)`
    - Audit: insert with action='activate_addon', note = feature name
    - revalidatePath both paths

    **deactivateAddon.ts** (per D-15):
    - Schema: `{ storeId: z.string().uuid(), feature: z.enum(['xero', 'email_notifications', 'custom_domain']) }`
    - Accepts FormData
    - Guard: query `store_plans` first. If override column is false, return `{ error: 'Cannot deactivate Stripe-managed add-on' }`
    - Write: `admin.from('store_plans').update({ [column]: false, [overrideColumn]: false }).eq('store_id', storeId)`
    - Audit: insert with action='deactivate_addon', note = feature name
    - revalidatePath both paths
