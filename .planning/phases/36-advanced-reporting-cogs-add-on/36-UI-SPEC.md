## Spacing Scale

Declared values — all multiples of 4, sourced from globals.css `--space-*` custom properties:

| Token | Value | CSS Variable | Usage |
|-------|-------|-------------|-------|
| xs | 4px | `var(--space-xs)` | Icon gaps, inline padding |
| sm | 8px | `var(--space-sm)` | Compact element spacing, gap between label and value |
| md | 16px | `var(--space-md)` | Default element spacing, card padding, form field gaps |
| lg | 24px | `var(--space-lg)` | Section padding, card interior padding |
| xl | 32px | `var(--space-xl)` | Layout gaps between major sections |
| 2xl | 48px | `var(--space-2xl)` | Major section breaks |
| 3xl | 64px | `var(--space-3xl)` | Page-level spacing |

Exceptions:
- Minimum touch target for any interactive control: 44x44px (POS rule, applies to chevron expand buttons on category rows).
- Summary card grid gap: `gap-3` (12px) matches existing `GSTSummaryBlock` pattern — permitted exception for dense card grids.
- Tab button bottom margin offset: `-mb-px` (negative 1px) to overlap parent border — matches existing `TabButton` component pattern.

Source: DESIGN.md + globals.css.

## Color

All tokens are CSS custom properties defined in `globals.css` (`@theme` block). Use Tailwind utility classes that map to these tokens.

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) | `--color-bg` | `#FAFAF9` | Page background — applies to the overall reports page |
| Secondary (30%) | `--color-surface` / `--color-card` | `#F5F5F4` / `#FFFFFF` | Summary cards use `bg-surface`. Table rows alternate `bg-card` / `bg-surface`. Table wrappers use `bg-card`. Category rows use `bg-card` on collapsed, `bg-surface` on expanded product sub-rows. |
| Accent (10%) | `--color-amber` | `#E67E22` | Reserved for: Export COGS CSV button only. Active tab underline uses `border-navy`, NOT amber. |
| Destructive | `--color-error` | `#DC2626` | Negative margin values (products selling below cost). Display as `text-error`. No destructive actions in this phase — no deletion UI. |
| Primary text | `--color-text` | `#1C1917` | All body text, table cell primary values |
| Muted text | `--color-text-muted` | `#78716C` | Secondary text, descriptions, empty state body, "—" null placeholders |
| Navy (interactive) | `--color-navy` | `#1E293B` | Table header background (`bg-navy text-white`). Active tab underline. Category row chevron icon. Section headings (`text-navy` = `text-primary`). |
| Border | `--color-border` | `#E7E5E4` | Table borders, card borders, tab nav bottom border |
| Success | `--color-success` | `#059669` | Positive margin % values above 30% threshold — `text-success`. Threshold: > 30% is healthy margin for NZ retail. |
| Warning | `--color-warning` | `#D97706` | Margin % values between 0% and 15% — `text-warning`. Signals low-margin products. |

Accent reserved for: "Export COGS CSV" button background only. No other elements in this phase use amber.

Margin % color coding:
- > 30%: `text-success` (#059669)
- 15%–30%: `text-primary` (#1C1917) — neutral
- 0%–15%: `text-warning` (#D97706)
- Negative: `text-error` (#DC2626)
- No cost price set ("—"): `text-muted` (#78716C)

Source: DESIGN.md color palette + confirmed by `GSTSummaryBlock.tsx` and `SalesSummaryTable.tsx`.

## Component Inventory

New components required for this phase. All follow existing codebase patterns.

### CogsReportSummaryCards

Layout: `grid grid-cols-2 md:grid-cols-4 gap-3`

4 cards in order:
1. **Revenue (excl. GST)** — `formatNZD(totalRevenueExclGstCents)` — `font-mono text-xl text-primary`
2. **Total Cost** — `formatNZD(totalCostCents)` — `font-mono text-xl text-primary`
3. **Gross Profit** — `formatNZD(totalMarginCents)` — `font-mono text-xl` + color-coded by margin direction (positive: `text-success`, negative: `text-error`, zero: `text-muted`)
4. **Margin %** — `{overallMarginPercent?.toFixed(1) ?? '—'}%` — `font-mono text-xl` + same color coding as Gross Profit card

Card shell: `bg-surface rounded-md px-4 py-4` — matches `GSTSummaryBlock` `SummaryCard`.
Label: `text-sm font-bold text-muted mb-1`.
Value: `font-mono text-xl`.

### CogsReportTable

Sortable per-product table. Columns: Product, Category, Units Sold, Revenue (excl. GST), Cost, Margin ($), Margin (%).

- Table wrapper: `overflow-x-auto rounded-lg border border-border`
- `<table className="w-full text-sm">`
- `<thead>`: `bg-navy text-white` — matches `SalesSummaryTable`
- Column headers: `px-4 py-3 font-bold` with sort indicator (chevron, 12px, inline SVG)
- Sortable columns: Units Sold, Revenue, Cost, Margin ($), Margin % — click header to sort ascending/descending
- Sort state: client-side `useState` — no server re-fetch on sort
- `<tbody>` rows: alternating `bg-card` / `bg-surface`, `border-t border-border`
- Text alignment: Product name left-aligned; all numeric columns right-aligned (`text-right font-mono`)
- Margin % cell: color-coded per margin % color coding rules above
- "—" for null margin (no cost price): `text-muted`
- `<tfoot>` totals row: `border-t-2 border-border bg-surface font-bold`

### CogsCategoryBreakdown

Collapsible category rows showing aggregated revenue/cost/margin per category with expandable product sub-rows.

- Table wrapper: `overflow-x-auto rounded-lg border border-border`
- Category row: `cursor-pointer hover:bg-surface bg-card border-t border-border`
- Category row left cell: `px-4 py-3 font-bold text-primary flex items-center gap-2`
- Chevron icon: inline SVG, 16x16, `stroke-current`, `transition-transform duration-150`, `rotate-90` when expanded
- Category name: `font-bold`
- Aggregate columns: Units Sold, Revenue (excl. GST), Cost, Margin ($), Margin % — right-aligned, `font-mono`
- Product sub-rows (when expanded): `bg-surface border-t border-border/50 text-sm`
- Product sub-row indent: `pl-10` on first cell (product name)
- Product name: `text-muted` (subordinate to category)
- Expand/collapse: client-side `useState<Set<string>>` — no server round-trip
- Transition: `duration-150 ease-out` on chevron rotation only — no row animation (matches "POS must feel instant" rule)
- All categories start collapsed

### ProductFormDrawer cost price field (conditional)

Only rendered when `hasAdvancedReporting === true`.

Position: directly below the selling price field in the product form.

```
Cost Price (excl. GST)    [NZ$  _____ ]
Supplier cost excluding GST. Used to calculate profit margin.
```

- Label: `text-sm font-semibold font-sans text-text`
- "(excl. GST)" annotation: `text-text-muted text-xs`
- Input: reuse existing `PriceInput` component pattern
- Helper text: `text-sm font-sans text-text-muted`

### ProductDataTable margin column (conditional)

Only rendered when `hasAdvancedReporting === true`.

New column appended after selling price: "Margin %"

- Header: `text-right font-bold px-4 py-3` (table header row)
- Cell value: formatted as `{margin.toFixed(1)}%` — color-coded per margin % rules above
- Null/no cost price: `text-muted` displaying "—"
- Column width: auto (does not require fixed width — numeric values are short)

### Profit & Margin tab button (conditional)

Rendered in `ReportsPageClient` tab nav only when `hasAdvancedReporting === true`.

Uses existing `TabButton` component (no new component needed):
```tsx
<TabButton href={...} active={activeTab === 'profit'}>
  Profit & Margin
</TabButton>
```

## Interaction States

### Tab navigation
- Inactive tab: `border-transparent text-muted hover:text-primary hover:border-border`
- Active tab: `border-navy text-navy` (navy underline — NOT amber)
- Transition: `transition-colors` (150ms, matches existing `TabButton`)

### Sort column headers (CogsReportTable)
- Default: chevron icon hidden or neutral color
- Ascending: chevron pointing up, `text-muted`
- Descending: chevron pointing down, `text-muted`
- Sort change: instant state update — no loading state

### Category row expand/collapse
- Collapsed: chevron pointing right (0deg)
- Expanded: chevron pointing right rotated 90deg (`rotate-90`)
- Transition: `transition-transform duration-150 ease-out` on chevron only
- Click target: full row width (`cursor-pointer`)

### Export COGS CSV button
- Default: `bg-amber text-white font-bold px-4 py-2 rounded-lg text-sm`
- Hover: `hover:bg-amber/90`
- Transition: `transition-colors` (150ms)
- Downloading: no loading state — client-side PapaParse is synchronous; download triggers immediately

### Cost price input in product form
- Default: standard `PriceInput` styling (matches selling price field)
- Optional field: no required indicator — empty/null is valid (D-03: no zero-cost assumption)

## Feature Gating — Visual Contract

Per CONTEXT.md decisions D-07, D-08, D-09: all COGS UI elements are **hidden** (not rendered, not disabled, not locked/teased) when `hasAdvancedReporting === false`.

| Element | When `hasAdvancedReporting === false` | When `hasAdvancedReporting === true` |
|---------|--------------------------------------|-------------------------------------|
| "Profit & Margin" tab | Not rendered — tab does not appear | Rendered as last tab in nav |
| Cost price field in product form | Not rendered | Rendered below selling price field |
| Margin % column in product list | Not rendered | Rendered as last column in product table |
| COGS summary cards | Not rendered | Rendered at top of Profit & Margin tab |
| COGS report tables | Not rendered | Rendered in Profit & Margin tab |

No upgrade prompts, lock icons, or teaser states on any gated element. Merchant discovers Advanced Reporting on the Billing page.

Source: CONTEXT.md D-07, D-08, D-09.
