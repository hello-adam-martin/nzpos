**src/app/api/xero/connect/route.ts** — OAuth initiation:
- GET handler. Import `XeroClient` from `xero-node`.
- Create XeroClient with `process.env.XERO_CLIENT_ID`, `process.env.XERO_CLIENT_SECRET`, `process.env.XERO_REDIRECT_URI`.
- Scopes: `['openid', 'profile', 'email', 'accounting.transactions', 'accounting.settings', 'offline_access']`.
- Call `xero.buildConsentUrl()` to get the consent URL.
- Return `Response.redirect(consentUrl)`.
- No auth check needed — the redirect itself goes to Xero, and the callback verifies the session.

**src/app/api/xero/callback/route.ts** — OAuth callback:
- GET handler. Receives authorization code from Xero redirect.
- Verify user is authenticated: use `createSupabaseServerClient()` to get user session. If no user, redirect to `/admin/integrations?error=not_authenticated`.
- Extract `store_id` from user's `app_metadata`.
- Create XeroClient (same config as connect route).
- Call `xero.apiCallback(url.toString())` to exchange code for tokens. This returns tokenSet.
- Call `xero.updateTenants()` to get tenant list.
- Extract `tenantId = xero.tenants[0].tenantId` and `tenantName = xero.tenants[0].tenantName`.
- Build token JSON: `{ access_token: tokenSet.access_token, refresh_token: tokenSet.refresh_token, expires_at: new Date(tokenSet.expires_at! * 1000).toISOString() }`.
- Call `saveXeroTokens(storeId, tokenJson)` to store in Vault. Get back secretId.
- Create or find Xero Contact "NZPOS Daily Sales": call `xero.accountingApi.getContacts(tenantId, undefined, 'Name=="NZPOS Daily Sales"')`. If not found, create via `xero.accountingApi.createContacts(tenantId, { contacts: [{ name: 'NZPOS Daily Sales' }] })`. Store contactID.
- Upsert `xero_connections` row via admin client: `store_id`, `tenant_id`, `tenant_name`, `vault_secret_id`, `status: 'connected'`, `xero_contact_id`.
- Redirect to `/admin/integrations?connected=true`.
- Wrap entire flow in try/catch. On error, redirect to `/admin/integrations?error=oauth_failed`.

**src/actions/xero/disconnectXero.ts** — Server Action:
- `'use server'` + `import 'server-only'`
- Verify owner auth via `createSupabaseServerClient()` and `getUser()`.
- Extract `store_id` from `app_metadata`.
- Optionally revoke token at Xero: call `getAuthenticatedXeroClient(storeId)`, then `xero.revokeToken()`. Catch and ignore errors (token may already be expired).
- Call `deleteXeroTokens(storeId)` to remove from Vault.
- Update `xero_connections` via admin client: set `status = 'disconnected'`, clear `account_code_cash`, `account_code_eftpos`, `account_code_online`, `xero_contact_id`.
- Call `revalidatePath('/admin')` to refresh layout banner.
- Return `{ success: true }`.

**src/actions/xero/saveXeroSettings.ts** — Server Action:
- `'use server'` + `import 'server-only'`
- Validate input with `XeroAccountCodesSchema.safeParse()`.
- Verify owner auth, extract `store_id`.
- Update `xero_connections` via admin client (service role to bypass RLS for simplicity, or use server client which has RLS passing since user is owner): set `account_code_cash`, `account_code_eftpos`, `account_code_online`.
- Call `revalidatePath('/admin/integrations')`.
- Return `{ success: true }` or `{ error: string }`.
