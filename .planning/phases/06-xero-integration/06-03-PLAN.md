**src/lib/xero/sync.ts** — Sync orchestration:

Import `server-only`. Import `createSupabaseAdminClient` from `@/lib/supabase/admin`. Import all xero lib modules.

**Internal helper: `aggregateDailySales(storeId: string, from: Date, to: Date): Promise<DailySalesData & { refundTotalCents: number }>`**
- Query orders via admin client: `.from('orders').select('total_cents, payment_method, channel, status').eq('store_id', storeId).gte('created_at', from.toISOString()).lte('created_at', to.toISOString())`
- Filter completed/collected/ready orders: `.in('status', ['completed', 'collected', 'ready'])`. Note: 'ready' is included because click-and-collect orders with status 'ready' have confirmed payment (paid online, awaiting pickup). Excluding them would undercount daily sales. 'pending_pickup' is NOT included as payment may not yet be confirmed.
- Aggregate:
  - `cashTotalCents`: sum of `total_cents` where `payment_method === 'cash'`
  - `eftposTotalCents`: sum of `total_cents` where `payment_method === 'eftpos'`
  - `onlineTotalCents`: sum of `total_cents` where `channel === 'online'`
- Also query refunded orders for the same period: `.eq('status', 'refunded')` — sum their `total_cents` as `refundTotalCents`.
- Return `{ cashTotalCents, eftposTotalCents, onlineTotalCents, dateLabel, refundTotalCents }`.

**Internal helper: `getXeroSettings(storeId: string): Promise<XeroSettings | null>`**
- Query `xero_connections` for `account_code_cash`, `account_code_eptpos`, `account_code_online`, `xero_contact_id` where `store_id = storeId` and `status = 'connected'`.
- Return null if no connection or any account code is null (not yet configured).
- Map to `XeroSettings` type.

**Internal helper: `writeSyncLog(entry: Partial<XeroSyncLogEntry> & { store_id: string; sync_date: string; sync_type: string; status: string }): Promise<string>`**
- Insert into `xero_sync_log` via admin client. Return the new row ID.

**Internal helper: `updateSyncLog(logId: string, updates: Partial<XeroSyncLogEntry>): Promise<void>`**
- Update `xero_sync_log` row by ID via admin client.

**Internal helper: `sleep(ms: number): Promise<void>`**
- Simple promise-based delay: `return new Promise(resolve => setTimeout(resolve, ms))`.

**Exported: `executeDailySync(storeId: string, referenceDate?: Date): Promise<{ success: boolean; message: string; invoiceNumber?: string }>`**
- Get authenticated Xero client. If null, return `{ success: false, message: 'Xero not connected' }`.
- Get Xero settings. If null, return `{ success: false, message: 'Account codes not configured' }`.
- Compute boundaries via `getNZDayBoundaries(referenceDate)`.
- Aggregate sales for the period.
- If all totals are zero and refundTotalCents is zero: return `{ success: true, message: 'No sales recorded for this period. Nothing to sync.' }`.
- Check for existing successful sync for this date: query `xero_sync_log` for `sync_date = label, status = 'success'`. If found, get `xero_invoice_id` for upsert.
- Write pending sync log entry.
- Build invoice via `buildDailyInvoice(sales, settings, existingInvoiceId)`.
- Try: call `xero.accountingApi.createInvoices(tenantId, { invoices: [invoice] })` (for new) or `xero.accountingApi.updateInvoice(tenantId, existingInvoiceId, { invoices: [invoice] })` (for upsert).
- Extract `invoiceID` and `invoiceNumber` from response: `result.body.invoices![0]`.
- If refundTotalCents > 0: build credit note via `buildCreditNote(refundTotalCents, label, settings, invoiceNumber)`, call `xero.accountingApi.createCreditNotes(tenantId, { creditNotes: [creditNote] })`.
- Update sync log to success with `xero_invoice_id`, `xero_invoice_number`, `total_cents` (sum of all payment method totals).
- Return `{ success: true, message: 'Synced', invoiceNumber }`.
- Catch: update sync log to failed with `error_message: error.message`. Return `{ success: false, message: error.message }`.

**Exported: `executeDailySyncWithRetry(storeId: string, referenceDate?: Date): Promise<{ success: boolean; message: string; invoiceNumber?: string; attempts: number }>`**
- Implements D-09: retry 3x with exponential backoff (1min, 5min, 15min) on sync failure.
- Backoff delays: `const BACKOFF_MS = [60_000, 300_000, 900_000]` (1 min, 5 min, 15 min).
- Maximum attempts: `const MAX_ATTEMPTS = 3`.
- Loop up to MAX_ATTEMPTS:
  1. Call `executeDailySync(storeId, referenceDate)`.
  2. If result.success is true: update the sync log entry's `attempt_count` to current attempt number, return `{ ...result, attempts: attemptNumber }`.
  3. If result.success is false AND this is not the last attempt:
     - Update sync log `attempt_count` to current attempt number.
     - Wait `BACKOFF_MS[attemptNumber - 1]` using `sleep()`.
     - Continue to next attempt.
  4. If result.success is false AND this is the last attempt:
     - Update sync log to `status: 'failed'`, `attempt_count: MAX_ATTEMPTS`, `error_message: result.message`.
     - Return `{ success: false, message: result.message, attempts: MAX_ATTEMPTS }`.
- Note: For non-retryable failures (Xero not connected, account codes not configured), return immediately without retrying — these are configuration issues, not transient errors. Only retry on Xero API errors.
- The cron route (Plan 04) calls `executeDailySyncWithRetry` instead of `executeDailySync` for automated overnight sync.

**Exported: `executeManualSync(storeId: string): Promise<{ success: boolean; message: string; invoiceNumber?: string }>`**
- Same logic as `executeDailySync` but uses `getNZTodayBoundaries()` instead of `getNZDayBoundaries()`.
- Sets `sync_type: 'manual'` in sync log.
- The upsert check looks for today's date in sync_log, so re-syncing today updates the existing invoice per D-10.
- Manual sync does NOT use retry logic — the owner can manually retry from the UI.
