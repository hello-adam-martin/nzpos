### ProductFormDrawer — product type radio group (NEW section, PROD-01)

```
+----------------------------------+
|  Product type                    |  ← heading, DM Sans 20px/600
|  ○ Physical  ● Service           |  ← radio buttons, side by side
+----------------------------------+
```

- Two radio buttons: "Physical" (left) and "Service" (right)
- Layout: horizontal flex, gap-sm (8px) between options
- Each option: white card (`--color-card`), border (`--color-border`), radius-md (8px), padding sm/md (8px 16px)

- Unselected state: surface background (`--color-surface`), border only
- "Physical" is pre-selected by default (D-02)
- Touch target: minimum 44px height on each option (iPad compliance)
- Source: D-01 (radio buttons side by side, always visible)

### ProductFormDrawer — conditional field visibility (D-03, D-05)

Two visibility rules are AND-combined:

| Condition | stock_quantity visible | reorder_threshold visible |
|-----------|----------------------|--------------------------|
| `hasInventory=true` AND `productType=physical` | YES | YES |
| `hasInventory=true` AND `productType=service` | NO | NO |
| `hasInventory=false` (any type) | NO | NO |

- Hidden fields: remove from DOM entirely (not `display:none`, not disabled/greyed)
- No transition animation on hide/show — instant removal per D-03
- Source: D-03, D-05

## Interaction States

### Product type radio (ProductFormDrawer)

| State | Visual |
|-------|--------|
| Default (physical selected) | Physical: amber ring + white bg. Service: surface bg + border only. |
| User clicks Service | Immediate swap: Service gets amber ring. stock/reorder fields disappear. |
| User clicks Physical | Immediate swap: Physical gets amber ring. stock/reorder fields appear (if hasInventory). |
| Disabled (form submitting) | Both options: opacity-50, cursor-not-allowed, ring stays on current selection |

No transition animation on the radio switch itself. Field show/hide is instant (D-03).

### CSV row error (ImportPreviewTable)

| State | Visual |
|-------|--------|
| Invalid product_type value | Row highlighted with red left border 3px (`--color-error`). Error message below row. Row marked "skipped". |
| Valid product_type | No decoration. Shows "Physical" or "Service" in the product_type column. |
| service + stock_quantity value | No error shown (silently ignored per D-10). stock_quantity column shows "—" |

## Implementation Notes for Executor

1. `hasInventory` prop flows: Server Component reads `user?.app_metadata?.inventory === true` and passes as `hasInventory: boolean` to all Client Components. Never fetch this client-side (RESEARCH.md Pattern 4, Pattern 5).

2. Radio button implementation: Use `<fieldset>` + `<legend>` for accessibility. Use `role="radiogroup"` if not using fieldset. Each option is a `<label>` wrapping a visually-hidden `<input type="radio">` — style the label card, not the input.

3. Tailwind v4 token references: Use CSS custom properties directly — `border-[--color-amber]`, `bg-[--color-card]`, `ring-[--color-amber]`. Do not use Tailwind v3 `@apply` style.

4. "Physical is default" copy: Show `--color-text-muted` hint text below the radio group: "New products default to Physical." Only show when creating, not when editing.

5. Focus ring: Use amber ring on keyboard focus for the radio options (`focus-visible:ring-2 focus-visible:ring-[--color-amber]`).

6. No animation: Do not add `transition` classes to the field show/hide. Instant DOM removal is intentional.
