| Token | Value | Usage |

| md | 16px (`--space-md`) | Card internal padding, form field gaps |

## Interaction Contracts

### Refresh Data Button

| State | Appearance | Copy |
|-------|------------|------|
| Idle (synced > 5 min ago) | Amber button, full opacity | "Refresh Data" |
| Loading (sync in progress) | Amber button, opacity 60%, spinner leading icon | "Syncing..." |
| Disabled (< 5 min cooldown) | Gray button, disabled, opacity 50% | "Available in 4:32" |
| Error (rate limit hit server-side) | Button returns to idle; inline error below | "Sync unavailable вҖ” try again shortly" |

Spinner: inline SVG `animate-spin`, 16px, white stroke. Positioned with `gap-2` inside button's flex row.

Page dimming during sync: wrap the stat cards, MRR chart, and add-on chart in a container with `transition-opacity duration-150`. Apply `opacity-50` class while sync is in progress. The `AnalyticsSyncControls` region stays at full opacity (user can see the loading state).

### Success Feedback

No floating toast library is available (none installed in codebase). Use an inline success banner that appears below the sync controls row and above the stat cards.

| State | Appearance | Duration |
|-------|------------|---------|
| Success banner | `bg-[var(--color-success)] bg-opacity-10 border border-[var(--color-success)] rounded-md px-4 py-2 text-sm text-[var(--color-success)]` | Auto-dismiss after 4 seconds via `setTimeout` + `useState` |
| Hidden | Display none (not rendered) | вҖ” |

Banner copy: "Data refreshed successfully." (see Copywriting Contract below).

### Countdown Timer

Client-side countdown using `useEffect` + `setInterval`. Ticks every 1 second. Derives `mm:ss` from `retryAfterSeconds` prop decremented each tick. Clears interval at 0. When countdown reaches 0: button re-enables, countdown hidden.

## Copywriting Contract

| Element | Copy | Notes |
|---------|------|-------|
| Page heading | "Analytics" | Matches existing placeholder heading |
| Primary CTA | "Refresh Data" | Verb + noun. Not "Sync" (too technical), not "Update" (ambiguous). |
| Loading CTA | "Syncing..." | Shows during in-flight request |
| Cooldown CTA | "Available in 4:32" | mm:ss countdown, no "Refresh" prefix |
| Stat card 1 label | "Current MRR" | All-caps avoided вҖ” sentence case throughout |
| Stat card 1 sub-label | "Annual plans normalised to monthly" | Explains the Г·12 rule without jargon |
| Stat card 2 label | "Monthly Churn" | Count of cancellations, not rate |
| Stat card 2 sub-label | "Cancelled this month" | Clarifies scope |
| Stat card 3 label | "Active Subscriptions" | Raw count |
| Stat card 3 sub-label | Leave empty (no subLabel prop) | Value speaks for itself |
| Last synced prefix | "Last synced:" | Lowercase after colon; e.g. "Last synced: 2 hours ago" |
| Last synced never | "Never synced" | When `last_synced_at` is null |
| MRR chart heading | "MRR Trend вҖ” Last 6 Months" | Em dash separator, matches SignupTrendChart style |
| Add-on chart heading | "Revenue by Add-on" | Short, clear |
| MRR empty state | "No MRR data yet. Run a sync to populate." | Tells user what to do |
| Add-on empty state | "No revenue data yet. Run a sync to populate." | Same pattern |
| Success banner | "Data refreshed successfully." | Past tense, period terminated |
| Rate limit error | "Sync unavailable вҖ” try again shortly." | Em dash, no countdown in this message (button shows countdown) |
| Server error | "Sync failed. Check your connection and try again." | Problem + action |

Destructive actions: None in this phase. The analytics page is read-only. Sync is additive (overwrites snapshot table). No user-facing destructive confirmation needed.
