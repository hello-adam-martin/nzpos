    1. **Current MRR:** Query `platform_analytics_snapshots` where status IN ('active', 'past_due') and snapshot_month = current month. Sum `mrr_cents`. Format: `(totalMrrCents / 100).toLocaleString('en-NZ', { style: 'currency', currency: 'NZD', minimumFractionDigits: 0, maximumFractionDigits: 0 })`.

    2. **Churn count:** Query `platform_analytics_snapshots` where status = 'canceled' AND canceled_at >= start of current month. Use `{ count: 'exact', head: true }`. Display as integer.

    3. **Active subscriptions:** Query `platform_analytics_snapshots` where status IN ('active', 'past_due', 'trialing') and snapshot_month = current month. Use `{ count: 'exact', head: true }`. Display as integer.

    4. **MRR trend (6 months):** Build array of last 6 month keys (`format(subMonths(new Date(), 5 - i), 'yyyy-MM')` for i 0..5). For each month, query sum of mrr_cents where status IN ('active', 'past_due') AND snapshot_month = monthKey. Build `{ month: format(subMonths(new Date(), 5 - i), 'MMM yyyy'), mrr: totalCents / 100 }[]` array. Pass to MrrTrendChart.

    5. **Add-on revenue breakdown:** Query `platform_analytics_snapshots` where status IN ('active', 'past_due') AND addon_type IS NOT NULL and snapshot_month = current month. Group by addon_type (in JS — Supabase JS client doesn't support GROUP BY). Build `{ name: displayName, mrr: sumCents / 100 }[]`. Display names: `{ xero: 'Xero', inventory: 'Inventory', email_notifications: 'Email Notifications', custom_domain: 'Custom Domain' }`.

    6. **Last synced timestamp:** Query `analytics_sync_metadata` where id = 1, get `last_synced_at`. Pass to AnalyticsSyncControls.

    Use `Promise.all` for all queries (parallel fetches).
