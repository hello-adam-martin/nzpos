## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge padding |
| sm | 8px | Compact element spacing, input inner padding |
| md | 16px | Default element spacing, card internal padding |
| lg | 24px | Section padding, wizard step gaps |
| xl | 32px | Card padding (AuthCard pattern), layout gaps |
| 2xl | 48px | Major section breaks (landing page section padding) |
| 3xl | 64px | Page-level spacing, landing page hero vertical padding |

Exceptions:
- Minimum touch target: 44x44px — all wizard CTA buttons, checklist items, color swatches (per DESIGN.md POS-specific rule, applied to wizard as touch-heavy interface)
- Color swatch size: 40x40px minimum, with 4px gap between swatches
- File drop zone: minimum 120px tall to be a clear drag target

Source: DESIGN.md spacing scale + POS touch target rule.

## Typography

| Role | Font | Size | Weight | Line Height | Usage |
|------|------|------|--------|-------------|-------|
| Body | DM Sans | 16px | 400 | 1.5 | Wizard help text, landing page body copy, checklist descriptions, sub-headlines |
| Label | DM Sans | 14px | 400 | 1.4 | Form labels, step indicators, button text, slug preview label, file upload constraints, placeholder hints, below-CTA notes, footer links |
| Heading | DM Sans | 20px | 700 | 1.3 | Wizard step titles, checklist banner heading, landing page section headings, final CTA heading |
| Display | Satoshi | 48px | 700 | 1.1 | Landing page hero headline (desktop). Scale to 28px on mobile via responsive modifier. |

Note: 4 declared sizes. Label (14px) covers all small utility text including captions and help notes. Display (48px) is used for the hero headline only; all other prominent headings use the Heading role (20px). The final CTA heading ("Ready to run a better shop?") uses Heading role at 20px — the navy section background provides visual prominence without requiring a larger size. Satoshi is reserved for the landing page hero headline only. All other text uses DM Sans.

Source: DESIGN.md typography scale + existing component inspection (SignupForm.tsx, AdminSidebar.tsx). Revised by checker: collapsed 6 sizes to 4, reduced 4 weights to 2.

Accent is NOT used for: informational links, nav items, form labels, passive states, or section backgrounds.

## Screens and Interaction States

### Screen A — Wizard: Step 1 — Store Name

Layout: Centered card (max-w-md, 448px), white background, shadow-md, rounded-lg. Reuse AuthCard shell pattern.

Step indicator: Horizontal row of 3 numbered circles at top of card. Active step = amber fill, navy text. Completed step = navy fill, white checkmark. Pending step = border-only, muted text. Connected by thin line.

Fields:
- Store Name (text input, required). Pre-filled from signup if available. Help text: "This appears on your storefront and receipts."
- Slug (display-only read field, not editable). Show as: `yourdomain.com/` + `[slug]` in monospace. Label: "Your store URL (set during signup, cannot be changed)". Text color: `var(--color-text-muted)`.

Primary CTA: "Save & Continue" — amber button, full width, min-height 44px.
Skip CTA: "Skip for now" — ghost/text link, 14px, muted color, centered below button.

### Screen B — Wizard: Step 2 — Logo + Brand Color

Layout: Same centered card as Step 1.

Logo upload:
- Dashed border drop zone (border-dashed, border-2, `var(--color-border)`), rounded-lg, min-height 120px.
- Center-aligned upload icon (SVG) + text: "Drop your logo here, or click to browse". Caption below: "PNG, JPG or SVG — max 2MB".
- After upload: show logo preview (max 80px tall, object-contain) with "Remove" text link (14px, muted, right-aligned).
- Upload state: spinner replaces drop zone icon, text: "Uploading...".
- Error state: border turns `var(--color-error)`, message below: "Upload failed — file may be too large or wrong format. Try again."

Color swatches:
- Section label: "Brand Colour" (NZ spelling).
- 8 curated swatches in a flex-wrap row. Each swatch: 40x40px circle, 4px gap. Colors (chosen for storefront viability):
  1. `#1E293B` (Navy — default, matches platform)
  2. `#0F766E` (Teal)
  3. `#0369A1` (Ocean blue)
  4. `#4F46E5` (Indigo)
  5. `#7C3AED` (Violet)
  6. `#BE185D` (Rose)
  7. `#15803D` (Forest green)
  8. `#B45309` (Warm amber/brown)
- Selected state: 3px amber ring around the swatch (`outline: 3px solid var(--color-amber); outline-offset: 2px`).
- Default: Navy swatch selected.
- Help text below swatches (14px, muted): "This colour appears on your storefront header."

Primary CTA: "Save & Continue" — amber, full-width.
Skip CTA: "Skip for now" — ghost link.

### Screen C — Wizard: Step 3 — First Product

Layout: Same centered card. Card max-w-md.

Fields (all optional but with contextual prompts):
- Product Name (text input). Placeholder: "e.g. Ceramic mug".
- Price (NZD, text input with "$" prefix). Placeholder: "0.00". Note below: "Price includes GST."
- Category (select dropdown). Options populated from existing categories. Placeholder: "Select a category".
- Product Image (compact upload, not full drop zone — 64px square preview area + "Add image" link button).

Primary CTA: "Add Product & Finish" — amber, full-width.
Skip CTA: "Skip — go to dashboard" — ghost link.

Completion: After Save or Skip on Step 3, show a brief full-card success state (250ms fade-in):
- Large checkmark icon (24px, success green).
- Heading: "Your store is ready."
- Body: "You can update all of these details anytime from your dashboard."
- Auto-redirect to `/admin/dashboard` after 1500ms (no button required, but show subtle "Taking you there..." text in muted).

### Screen D — Dashboard: Setup Checklist Banner

Placement: Immediately below the admin dashboard top bar, full width of main content area. Above all other dashboard content.

Visual: White card background (`bg-white`), `border border-[var(--color-border)]`, `shadow-sm`, `rounded-lg`, padding `lg` (24px). No amber/colored background — it is informational, not urgent.

Content layout (horizontal on desktop, stacked on mobile):
- Left: Progress fraction label "2 of 5 steps complete" (20px, DM Sans 700, `var(--color-text)`).
- Center: Horizontal progress bar — full width fill strip, `var(--color-surface)` track, `var(--color-amber)` fill, height 8px, rounded-full. Fills proportionally to completed items.
- Right: "Resume setup" link button (14px, navy, underline on hover) — links to `/admin/setup` at the first incomplete step.

Checklist items: Vertical list below the progress row. Each item:
- Incomplete: circle checkbox (outline, 20px), label text in `var(--color-text)`.
- Complete: filled checkmark circle (`var(--color-success)`), label text in `var(--color-text-muted)` with line-through.

5 items:
1. Set your store name
2. Upload your logo
3. Add your first product
4. Complete your first POS sale
5. Receive your first online order

Completion state: When all 5 items complete, replace the full banner content with:
- Centered text (20px, DM Sans 700): "Your store is set up."
- Body (16px, muted): "You're all set — everything is ready to go."
- Duration: visible for 3 seconds (CSS animation: opacity 1 → 0 after 3s delay), then banner collapses to height 0 with `transition: max-height 400ms ease-out`. Banner never reappears (DB flag set).

### Screen E — Landing Page (root domain `/`)

Layout: Full-bleed sections, max-content-width 1200px centered, horizontal padding 24px (mobile: 16px).

Static rendering: `export const dynamic = 'force-static'`. No client-side data fetching.

Sections in order:

**Nav bar:**
- Background: `var(--color-navy)`. Sticky top.
- Left: "NZPOS" wordmark in Satoshi 700, white.
- Right: "Sign in" ghost button (white border) + "Get started" amber button. On mobile: hamburger → full-screen overlay.

**Hero section:**
- Full-bleed navy background (`var(--color-navy-dark)`).
- Vertical padding: 64px top, 64px bottom (desktop). 48px top/bottom mobile.
- Headline: Display role — Satoshi 48px / 700 / lh 1.1, white. Mobile: 28px via responsive modifier.
  Copy: "The POS built for Kiwi retailers."
- Sub-headline: DM Sans 16px / 400 / lh 1.5, `rgba(255,255,255,0.7)`.
  Copy: "Ring up sales in-store, take orders online, and get GST handled correctly — all from one dashboard."
- CTA button: "Get started free" — amber (`var(--color-amber)`), padding `sm` `xl` (8px 32px), rounded-md, DM Sans 14px 700. On click: navigate to `/signup`.
- Below button: "No credit card required. NZD pricing." — 14px, `rgba(255,255,255,0.5)`.
- Hero illustration/image: Right-aligned on desktop (50% width). Use a stylised iPad mockup showing the POS interface, or a clean screenshot. On mobile: hidden (content-only hero).

**Features section:**
- Background: `var(--color-bg)` (warm stone).
- Heading: "Everything a Kiwi shop needs" — DM Sans 20px / 700, `var(--color-text)`, centered.
- 3-column grid (desktop), 1-column (mobile). Each feature card: white bg, border, shadow-sm, rounded-lg, padding 24px.
- 4 feature cards:
  1. Icon + "POS on iPad" — "Tap to sell. Scan barcodes. EFTPOS confirmed manually."
  2. Icon + "Online store included" — "Your storefront is live from day one. Click-and-collect ready."
  3. Icon + "GST done right" — "15% GST calculated per line, IRD-compliant on every receipt."
  4. Icon + "Stock stays in sync" — "Sell in-store or online — inventory updates instantly."
- Icons: inline SVG, 24x24, `var(--color-navy)`.

**Pricing section:**
- Background: `var(--color-surface)`.
- Heading: "Transparent pricing" — DM Sans 20px / 700, centered.
- Sub-heading: "Free to start. Pay only for what you use." — 16px, muted, centered.
- Layout: 1 primary card (Free core) + 2 add-on pills side by side.
- Free core card: white bg, border-2 `var(--color-navy)`, rounded-lg, padding 32px. Prominent "Free" label in Satoshi Display role — 48px / 700, responsive to 28px mobile.
  Includes: POS checkout, online storefront, inventory management, GST-compliant receipts.
- Add-on pills: `var(--color-card)`, border, rounded-lg, padding 24px.
  - Xero Integration: "$9/month NZD"
  - Email Notifications: "$5/month NZD"
  - Each shows what it includes in 2-3 bullet points.
- Note below: "All prices include GST. No hidden fees." — 14px, muted.

**Final CTA section:**
- Background: `var(--color-navy)`.
- Heading: "Ready to run a better shop?" — DM Sans 20px / 700, white. Visual prominence from the dark navy section background.
- CTA button: "Get started free" — amber button (same as hero).
- Sub-text: "Set up in under 5 minutes." — 14px, `rgba(255,255,255,0.6)`.

**Footer:**
- Background: `var(--color-navy-dark)`.
- Copyright: "© 2026 NZPOS. Built in New Zealand." — 14px, `rgba(255,255,255,0.4)`.
- Links: "Sign in" | "Privacy" | "Terms" — 14px, `rgba(255,255,255,0.5)`, hover white.

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Wizard step indicator | `aria-label="Step {N} of 3"` on each dot, `aria-current="step"` on active |
| Color swatches | Each swatch is a `<button>` with `aria-label="{color name}"` and `aria-pressed={selected}` |
| Logo drop zone | `role="button"` with `aria-label="Upload logo"`, keyboard activatable |
| File input | Hidden `<input type="file">` triggered by the drop zone button |
| Checklist items | `<ul>` with `<li>` items, completed items have `aria-label="{item} — complete"` |
| Landing page CTAs | Both "Get started free" buttons have unique `aria-label` values (hero vs final CTA section) |
| Progress bar | `role="progressbar"`, `aria-valuenow`, `aria-valuemin="0"`, `aria-valuemax="5"` |
| Form errors | All field errors use `aria-describedby` linking input to error message |
| Skip links | Wizard skip links are visually prominent (not visually hidden) per accessible design |

Focus management: After wizard step save, focus moves to the step indicator heading of the next step (`autoFocus` on the new step's `h2`).
