## Spacing Scale

Declared values (all multiples of 4 — using CSS custom properties matching existing codebase):

| Token | Value | CSS Var | Usage |
|-------|-------|---------|-------|
| 2xs | 2px | — | Micro gaps within inline elements |
| xs | 4px | `--space-xs` | Icon gaps, tight inline padding |
| sm | 8px | `--space-sm` | Compact element spacing, badge padding |
| md | 16px | `--space-md` | Default element spacing, card internal padding |
| lg | 24px | `--space-lg` | Section padding, between cards |
| xl | 32px | `--space-xl` | Chart internal padding (matches SalesTrendChart) |
| 2xl | 48px | `--space-2xl` | Major section breaks |
| 3xl | 64px | `--space-3xl` | Page-level vertical rhythm |

Exceptions:

- Payment failure warning banner uses `py-3 px-4` (12px / 16px) — compact alert, not a card
- Action button minimum touch target: 36px height on desktop super-admin UI (not POS, no 44px rule)
- Sidebar nav links: `px-3 py-3` (matches existing SuperAdminSidebar pattern exactly)

## Component Specifications

### Dashboard Stat Cards (SA-DASH-01, SA-DASH-02)

Reuse `DashboardHeroCard` pattern exactly:
- Container: `bg-white border border-[var(--color-border)] shadow-sm rounded-lg p-[var(--space-md)] min-h-[96px] flex flex-col justify-between`
- Label: `font-sans font-bold text-sm text-[var(--color-text-muted)]`
- Value: `font-display font-bold text-3xl text-[var(--color-navy)]` with `fontFeatureSettings: "'tnum' 1"`
- Sub-label: `font-sans font-normal text-sm text-[var(--color-text-muted)] mt-1`

Grid layout for stat cards: `grid grid-cols-2 gap-4 md:grid-cols-4` — 4 cards on desktop (active tenants, suspended tenants, new signups this month, and one card per add-on showing adoption %).

Add-on adoption badges within stat cards: small pill inside the card sub-label area — `inline-flex items-center px-2 py-0.5 text-xs font-semibold rounded-full bg-[var(--color-amber)]/10 text-[var(--color-amber)]` showing "X% adoption".

### Signup Trend Chart (SA-DASH-03)

Reuse `SalesTrendChart` visual pattern (AreaChart) with these differences:
- dataKey is signup count (integer), not currency cents
- Y-axis formatter: plain integer (no `$` prefix)
- Stroke: `#1E293B` (navy) — same as SalesTrendChart
- Area fill gradient: amber (`#E67E22`) at 15% opacity top, 0% bottom — matches SalesTrendChart gradient pattern
- Chart height: 200px (slightly shorter — no period toggle needed, fixed 30-day view)
- X-axis: day labels ("1 Apr", "2 Apr", …) at 14px DM Sans
- No period toggle — always 30-day window
- Empty state: `<p className="text-sm text-[var(--color-text-muted)]">No signups in the last 30 days.</p>` centered in chart area at py-8

Chart card container: `bg-card border border-[var(--color-border)] shadow-sm rounded-[var(--radius-lg)] p-[var(--space-xl)]` — matches SalesTrendChart wrapper exactly.

### Payment Failure Warning Banner (D-07)

Shown at the very top of the tenant detail page body when any invoice has `status: past_due` or `status: open` AND the invoice is overdue.

```
bg-[var(--color-warning)]/10 border border-[var(--color-warning)]/30 rounded-[var(--radius-md)] p-4 mb-6
```

Structure: icon (warning triangle, 16x16, `text-[var(--color-warning)]`) + text inline.
- Heading line: `text-sm font-semibold font-sans text-[var(--color-text)]` — "Payment overdue"
- Body line: `text-sm font-sans text-[var(--color-text-muted)]` — "This tenant has a past-due invoice. Their account may be restricted."

This matches the SuspendModal warning box pattern (`bg-[var(--color-amber)]/10 border border-[var(--color-amber)]/30`) but uses warning amber (darker: `#D97706`) instead of accent amber.

### Stripe Subscriptions Section Card

New card in left column (col-span-2) on tenant detail page, below Add-ons card.

Card header: `font-display text-xl font-semibold text-[var(--color-text)] mb-4` — "Subscriptions"

Each subscription as a row:
- Plan name: `text-sm font-semibold font-sans text-[var(--color-text)]`
- Status badge: `inline-flex items-center px-2 py-0.5 text-xs font-semibold rounded-full` — active=`bg-[var(--color-success)]/10 text-[var(--color-success)]`, canceled=`bg-[var(--color-error)]/10 text-[var(--color-error)]`
- Amount + interval: `font-mono text-sm text-[var(--color-text-muted)]` (e.g. "$29.00 / month")
- Period end date: `text-sm font-sans text-[var(--color-text-muted)]` (e.g. "Renews 12 May 2026")

Loading state: Three skeleton rows using `animate-pulse bg-[var(--color-border)] rounded h-4 w-full` at heights matching row content — 3 rows at 40px each.

Empty state (no subscriptions): `text-sm font-sans text-[var(--color-text-muted)]` — "No active subscriptions found for this tenant."

### Stripe Invoices Section Card

New card in left column (col-span-2) on tenant detail page, below Subscriptions card.

Card header: "Recent Invoices" (same heading style).

Invoices as a table:
- Table: `w-full text-sm font-sans`
- Header row: `text-xs font-semibold font-sans text-[var(--color-text-muted)] uppercase tracking-wider border-b border-[var(--color-border)] pb-2`
- Columns: Date | Description | Amount | Status
- Row cells: `py-3 border-b border-[var(--color-border-light)] last:border-0`
- Date: `font-sans text-sm text-[var(--color-text-muted)]` — "d MMM yyyy" format
- Description: `font-sans text-sm text-[var(--color-text)]`
- Amount: `font-mono text-sm text-[var(--color-text)]` with tabular-nums
- Status badge:
  - paid: `bg-[var(--color-success)]/10 text-[var(--color-success)]`
  - open/pending: `bg-[var(--color-info)]/10 text-[var(--color-info)]`
  - past_due/void: `bg-[var(--color-error)]/10 text-[var(--color-error)]`
  - All badges: `inline-flex items-center px-2 py-0.5 text-xs font-semibold rounded-full`

Loading state: Skeleton table — 5 rows of `animate-pulse bg-[var(--color-border)] rounded h-4` at varying widths.

Empty state: `text-sm font-sans text-[var(--color-text-muted)]` — "No invoices found for this tenant."

### Password Reset Modal (D-08)

Reuse SuspendModal structure exactly:
- Overlay: `fixed inset-0 bg-black/50 z-50 flex items-center justify-center`
- Panel: `bg-white rounded-[var(--radius-lg)] shadow-lg p-6 max-w-md w-full mx-4`
- Title: `font-display text-xl font-semibold text-[var(--color-text)] mb-4` — "Send password reset?"
- Info box (not warning): `bg-[var(--color-info)]/10 border border-[var(--color-info)]/30 rounded-[var(--radius-md)] p-4 mb-4` — "A password reset email will be sent to [email]. The owner will receive a link to set a new password."
- No form input required — action is immediate on confirmation
- Confirm button: navy primary — `bg-[var(--color-navy)] text-white text-sm font-semibold px-4 py-2 rounded-[var(--radius-md)] hover:bg-[var(--color-navy)]/90`
- Cancel: ghost text button — `text-sm font-semibold font-sans text-[var(--color-text-muted)] hover:text-[var(--color-text)]`
- Pending state: spinner + "Sending…" label (matches SuspendModal ConfirmButton pattern)

### Disable Account Modal (D-09)

Reuse SuspendModal structure exactly:
- Title: "Disable [StoreName]?"
- Warning box: amber (`bg-[var(--color-amber)]/10 border border-[var(--color-amber)]/30`) — "This will ban the owner from logging in and suspend the store. The storefront and POS will become inaccessible."
- Confirm button: destructive — `bg-[var(--color-error)] text-white` (matches SuspendModal ConfirmButton exactly)
- Label: "Disable Account"
- No reason field required (unlike suspend which requires reason text)
- Pending state: spinner + "Disabling…"

Re-enable modal (reverse flow):
- Title: "Re-enable [StoreName]?"
- Info box (blue): "This will restore login access and unsuspend the store."
- Confirm button: navy primary (not destructive)
- Label: "Re-enable Account"

### Updated Sidebar Nav (D-11)

Three nav links in order:
1. Dashboard — `/super-admin`
2. Tenants — `/super-admin/tenants`
3. Analytics — `/super-admin/analytics` (placeholder, greyed-out treatment)

Analytics link treatment: same `text-white/60 hover:text-white hover:bg-white/5` style as inactive links. No "coming soon" tooltip — just a link that lands on a stub page.

Sidebar active state logic: use `pathname === href || pathname.startsWith(href + '/')` — but `/super-admin` must use exact match only (`pathname === '/super-admin'`) to avoid all three lighting up when on sub-pages.

### Analytics Placeholder Page

Minimal stub page at `/super-admin/analytics`:
- Page heading: `font-display text-2xl font-semibold text-[var(--color-text)]` — "Analytics"
- Single card: `bg-white border border-[var(--color-border)] shadow-sm rounded-lg p-8 text-center`
- Content: `text-sm font-sans text-[var(--color-text-muted)]` — "Platform analytics are coming in Phase 27. MRR, churn, and revenue breakdown will appear here."

## Interaction States

| Component | Idle | Hover | Loading | Error | Empty |
|-----------|------|-------|---------|-------|-------|
| Stat cards | white card, navy value | no hover effect (static) | skeleton pulse | — | "0" value shown |
| Signup trend chart | area chart rendered | tooltip on point | skeleton card (same height) | "Could not load data." text | "No signups" message |
| Subscription rows | standard row | no row hover (read-only list) | skeleton 3 rows | error text | empty state text |
| Invoice table | standard rows | no row hover (read-only) | skeleton 5 rows | error text | empty state text |
| Warning banner | visible when past-due | — | — | — | hidden (not rendered) |
| "Send Password Reset" button | navy, enabled | navy/90 | disabled + spinner | inline error | — |
| "Disable Account" button | navy outline or ghost, enabled | hover tint | disabled + spinner | inline error | — |
| "Re-enable Account" button | navy outline or ghost | hover tint | disabled + spinner | inline error | — |
| Modal overlay | visible | click outside closes | — | inline error in panel | — |

Loading skeleton pattern: `animate-pulse bg-[var(--color-border)] rounded` — matches project convention. Duration uses Tailwind default (1.5s). No spinner at section level — skeletons only.

Motion: all transitions at 150ms ease-out (`transition-colors duration-150`) matching existing super-admin components.
