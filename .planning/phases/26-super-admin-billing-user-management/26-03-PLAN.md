    - `'use server'` + `import 'server-only'`
    - Zod schema: `{ storeId: z.string().uuid(), ownerAuthId: z.string().uuid() }`
    - Auth check same as above
    - Step 1: Fetch store slug for cache invalidation: `admin.from('stores').select('slug').eq('id', storeId).single()`
    - Step 2: Ban user in Supabase Auth: `admin.auth.admin.updateUserById(ownerAuthId, { ban_duration: '876600h' })`
    - Step 3: Suspend store: `admin.from('stores').update({ is_active: false, suspended_at: new Date().toISOString(), suspension_reason: 'Account disabled by super-admin' }).eq('id', storeId)`
    - Step 4: Invalidate cache: `invalidateCachedStoreId(store.slug)` (import from `@/lib/tenantCache`)
    - Step 5: Audit log: `action: 'disable_account'`, note: `'Account disabled and owner banned'`
    - Step 6: Revalidate paths: `/super-admin/tenants` and `/super-admin/tenants/${storeId}`
    - Return type: `Promise<{ success: true } | { error: string }>`

    **enableMerchantAccount.ts (SA-USER-03 reverse):** Create following unsuspendTenant.ts pattern:
    - `'use server'` + `import 'server-only'`
    - Zod schema: `{ storeId: z.string().uuid(), ownerAuthId: z.string().uuid() }`
    - Auth check same as above
    - Step 1: Fetch store slug for cache invalidation
    - Step 2: Unban user: `admin.auth.admin.updateUserById(ownerAuthId, { ban_duration: 'none' })`
    - Step 3: Unsuspend store: `admin.from('stores').update({ is_active: true, suspended_at: null, suspension_reason: null }).eq('id', storeId)`
    - Step 4: `invalidateCachedStoreId(store.slug)`
    - Step 5: Audit log: `action: 'enable_account'`, note: `'Account re-enabled and owner unbanned'`
    - Step 6: Revalidate both paths
    - Return type: `Promise<{ success: true } | { error: string }>`
