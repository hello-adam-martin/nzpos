| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline spacing |
| sm | 8px | Input padding (vertical), label-to-input gap |
| md | 16px | Card internal padding, form field spacing |
| lg | 24px | Card external padding, section gaps, page padding (mobile) |
| xl | 32px | Page section breaks |
| 2xl | 48px | Page top/bottom padding (desktop) |

Exceptions:
- Account icon touch target: 40×40px (matches existing cart icon: `h-10 w-10`)
- Dropdown menu item height: 40px minimum (finger-safe for mobile)

Source: DESIGN.md spacing scale + existing StorefrontHeader.tsx `h-10 w-10` cart icon pattern.

## Typography

Phase 10 declares 4 sizes and 2 weights. The Display role (Satoshi 20px) is inherited from the pre-existing StorefrontHeader and is not new to this phase — it is not redeclared here.

| Role | Font | Size | Weight | Line Height | Notes |
|------|------|------|--------|-------------|-------|
| Heading | DM Sans (`--font-sans`) | 24px | 600 | 1.2 | Page titles (My Orders, My Profile, Sign in) |
| Body | DM Sans (`--font-sans`) | 16px | 400 | 1.5 | Paragraph text, form descriptions |
| Data | Geist Mono (`--font-mono`) | 16px | 400 | 1.0 | Order numbers (#ABC12345) |
| Label | DM Sans (`--font-sans`) | 14px | 600 | 1.4 | Form labels, nav items, metadata |
| Small | DM Sans (`--font-sans`) | 12px | 400 | 1.5 | Timestamps, status metadata, muted annotations |

Weights: 400 (regular) and 600 (semibold) only. No weight 700 in Phase 10 components — all bold/semibold text uses 600. This matches the existing login page pattern (`font-semibold` = 600).

Source: DESIGN.md typography scale + existing login page (`text-2xl font-semibold` heading, `text-sm` labels) + confirmation page (`font-mono text-lg` for order numbers).

## Interaction Contracts

### Account Dropdown
- **Open trigger:** Click on `AccountMenuButton` (not hover — mobile-safe)
- **Close trigger:** Click outside, press Escape, or click any menu item
- **Animation:** Fade + scale-in from top-right, 150ms ease-out. CSS `opacity-0 scale-95` → `opacity-100 scale-100`.
- **Z-index:** z-50 (same as header)
- **No nested interactions** — flat list only

### Verification Banner
- **Persistent:** Shown on every page render until customer's `email_confirmed_at` is non-null
- **Resend action:** Inline `<button>` styled as text link. On click: disable button for 30s, show "Sent!" inline, re-enable. Server Action calls Supabase `resend` confirmation.
- **No dismiss:** Cannot be dismissed manually. Disappears when email confirmed.

### Order History
- **Loading state:** Skeleton cards — 3 placeholder cards with `animate-pulse bg-surface rounded-lg h-20`.
- **Pagination:** None for v1. Show all orders, newest first. If more than 20 orders, implement simple "Load more" button (not pagination nav).
- **Empty state:** See Copywriting section below.

### Profile Form
- **Save behavior:** Server Action. Optimistic UI — button shows "Saving…" during submit. On success: inline "Saved" confirmation text (emerald, 2s timeout, then disappears). On error: inline error message below affected field.
- **Email change:** If email changed, trigger re-verification. Show inline note: "A verification email will be sent to your new address." after save.
- **Password change:** Collapsible section. Click "Change password" text link to expand. Fields: Current password, New password, Confirm new password. Submit independently from profile save.

### Post-Purchase Prompt
- **Shown when:** Order confirmation page rendered AND no customer session
- **Dismiss:** "No thanks" plain text link. Hides prompt for the session (localStorage key `dismissed_account_prompt`). Does not affect page layout — just removes the card.
- **CTA click:** Links to `/account/signup?return_to=/order/{id}?token={token}` so customer can return to their order after signup.

### Form Validation
- **Inline errors:** Below each field, `text-sm text-error`, appear on blur (not on submit)
- **Submit error:** Full-width alert above submit button. Red tinted (`bg-red-50 border border-red-200 text-error text-sm rounded-md p-3`)
- **Focus ring:** `focus:outline-none focus:ring-2 focus:ring-navy/20 focus:border-navy/40` (matches existing login page)

## Accessibility Constraints

- All interactive elements have `aria-label` or visible text label
- `AccountMenuButton`: `aria-label="Account menu"` when logged out, `aria-label="Account, {name}"` when logged in
- `AccountDropdown`: `role="menu"`, items `role="menuitem"`, Escape key closes
- `PreferenceToggle`: `role="switch"`, `aria-checked={checked}`, `aria-label="{preference name}"`
- `VerificationBanner`: `role="alert"` so screen readers announce it on page load
- `OrderHistoryCard`: anchor wrapping the card — no nested interactive elements inside (except separate action buttons if any are added later)
- Form inputs: each has `id` + `htmlFor` pairing, matching the existing login page pattern
- Password inputs: `autoComplete="current-password"` or `autoComplete="new-password"` as appropriate
