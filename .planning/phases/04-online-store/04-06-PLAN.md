**Allowed transitions map (per D-22):**
```typescript
const ALLOWED_TRANSITIONS: Record<string, string[]> = {
  'completed': ['pending_pickup'],
  'pending_pickup': ['ready'],
  'ready': ['collected'],
}
```

**Input validation with Zod:**
```typescript
const UpdateStatusSchema = z.object({
  orderId: z.string().uuid(),
  newStatus: z.enum(['pending_pickup', 'ready', 'collected']),
})
```

**Logic:**
1. Auth check: Get user from Supabase auth. Extract store_id from app_metadata. If not authenticated, return `{ error: 'Not authenticated' }`.
2. Validate input with UpdateStatusSchema.
3. Fetch current order: `.select('id, status, store_id, channel').eq('id', orderId).eq('store_id', storeId).single()`
4. Check order exists and belongs to store.
5. Validate transition: Check `ALLOWED_TRANSITIONS[currentStatus]?.includes(newStatus)`. If not allowed, return `{ error: 'Invalid status transition from {currentStatus} to {newStatus}' }`.
6. Update order status: `.update({ status: newStatus }).eq('id', orderId)`
7. If transitioning to 'ready' (D-14): This is where email notification would be sent. For now, add a TODO comment: `// TODO: Send pickup-ready email notification (Resend integration)`. The actual email sending will be wired when Resend is integrated.
8. `revalidatePath('/pos/pickups')` and `revalidatePath('/admin/orders')` (for Phase 5 admin).
9. Return `{ success: true, newStatus }`.
