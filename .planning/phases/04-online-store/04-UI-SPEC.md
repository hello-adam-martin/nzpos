## Spacing Scale

Declared values (multiples of 4, sourced from DESIGN.md):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, chip internal padding |
| sm | 8px | Compact element spacing, card internal gaps |
| md | 16px | Default element spacing, input padding, button padding |
| lg | 24px | Section padding, card padding |
| xl | 32px | Layout gaps, content section breaks |
| 2xl | 48px | Major section breaks, hero vertical padding |
| 3xl | 64px | Page-level spacing, above-fold hero height supplement |

Exceptions:
- Touch targets on mobile: minimum 44x44px (inherited from DESIGN.md POS rule; applies to storefront Add to Cart button on mobile)
- Product card minimum width: 160px on desktop, 140px on mobile
- Max content width: 1200px centered (storefront specific, from DESIGN.md)
- Storefront header height: 64px fixed
- Fine separators (2px): inherited from DESIGN.md spacing scale (`2xs: 2px`). Achieved via 1px border width rather than a spacing token where possible. Not a standard spacing step — use only for decorative dividers with no layout spacing responsibility.

## Typography

Sourced from DESIGN.md. Storefront uses exactly 4 roles: Body, Label, Heading, Display.

Note: This phase uses 2 weights — 400 (regular) and 600 (semibold). These are a subset of the full DESIGN.md weight range (400–700). The full range is available in globals.css but only 400 and 600 are in use for this phase to maintain visual hierarchy with minimal weight variation.

| Role | Font | Size | Weight | Line Height | Usage |
|------|------|------|--------|-------------|-------|
| Body | DM Sans | 16px (base) | 400 | 1.5 | Product descriptions, cart summary text, footer copy, product name on card (weight 600 for differentiation), price on product list card (weight 600), cart total, checkout summary total |
| Label | DM Sans | 14px (sm) | 600 | 1.4 | Category pills, form labels, badge text, nav links, stock badge labels, timestamp/meta text, legal footnotes |
| Heading | DM Sans | 24px (2xl) | 600 | 1.3 | Product names on detail page, section headings |
| Display | Satoshi | 36px (4xl) | 600 | 1.2 | Store name/hero headline, page-level titles |

Notes on role assignment:
- Label weight changed from 500 to 600 to conform to the 2-weight constraint. At 14px, 600 reads clearly as a label without competing with body text.
- Display weight changed from 700 to 600. Satoshi at 600 and 36px still reads as a strong display-weight heading. Full 700 is available if a future phase requires stronger hero contrast.
- Badge/meta text (previously 12px) uses Label (14px) — minimum legible size for touch devices.
- Product card name and price (previously 18px) use Body (16px) at weight 600 for visual differentiation from description text.
- Cart total and checkout summary total (previously 20px) use Body (16px) at weight 600; the larger Heading role (24px) is reserved for section headings on the detail page.

Price figures: DM Sans, `tabular-nums` feature enabled (`font-feature-settings: 'tnum' 1`), weight 600.

## Color

Sourced from globals.css @theme tokens. The 60/30/10 rule:

| Role | Value | Token | Usage |
|------|-------|-------|-------|
| Dominant (60%) | #FAFAF9 | --color-bg | Page background, main content area background |
| Secondary (30%) | #F5F5F4 | --color-surface | Product card backgrounds, nav bar background, sidebar, footer |
| Accent (10%) | #E67E22 | --color-amber | Add to Cart button, Checkout button, active promo badge, price highlight on sale items |
| Destructive | #DC2626 | --color-error | Remove from cart action, promo code error state, sold out badge |

White (#FFFFFF / --color-card) used for elevated surfaces: product image containers, modals, dropdown menus.

Accent reserved for EXACTLY these elements:
1. "Add to Cart" button (primary CTA on product card and detail page)
2. "Proceed to Checkout" button in cart drawer
3. Active/selected category pill fill
4. Promo code applied success badge background (tinted, not full amber)

Accent is NOT used for: navigation links, headings, body text, form inputs, borders, or hover states on non-CTA elements.

Secondary semantic colors:
- Success #059669 (--color-success): Promo applied confirmation, "In Stock" badge
- Warning #D97706 (--color-warning): Low stock notice ("Only 3 left")
- Info #3B82F6 (--color-info): Order status informational chip (PENDING_PICKUP)
- Error #DC2626 (--color-error): Sold Out badge, promo code invalid message, form errors

## Layout

Sourced from DESIGN.md layout section:

### Storefront Global Layout
- Header: sticky top, 64px height, contains store logo (left), cart icon + item count (right), category nav (below header or inline)
- Content: max-width 1200px, horizontally centered, px-6 on mobile → px-8 on desktop
- Footer: --color-surface background, store name + "Powered by NZPOS" tagline, no navigation complexity

### Product Listing Page
- Mobile: 2-column product grid, 8px gap
- Tablet (≥768px): 3-column product grid, 16px gap
- Desktop (≥1024px): 4-column product grid, 16px gap
- Category filter: horizontal scrollable pill row, sticky below header

### Product Detail Page
- Mobile: single column — image top, details below
- Desktop: 2-column split — image left (50%), details right (50%)
- Image: square aspect ratio, rounded-lg (8px), object-cover
- Details: product name (heading role), price (body role at weight 600, amber), description (body), Add to Cart CTA

### Cart
- Desktop: side drawer panel sliding from right, 380px wide
- Mobile: full-screen bottom sheet modal
- Cart is empty: centered empty state illustration area + CTA to continue shopping
- Cart has items: line item list + promo code input + order summary + Checkout CTA

### Checkout
- Redirect to Stripe Checkout hosted page (no custom checkout UI to design)
- Post-checkout: Stripe redirects to /store/order-confirmation?session_id={id}

### Order Confirmation Page
- Centered, max 600px
- Success icon (emerald green, --color-success)
- Order number (Geist Mono, --font-mono)
- Summary: items, total, GST breakdown
- Click-and-collect status message

| Interaction | Duration | Easing | Notes |
|-------------|----------|--------|-------|
| Cart drawer open/close | 250ms | ease-in-out | Slide from right |
| Product card hover | 150ms | ease-out | Shadow lift only, no transform |
| Add to Cart button press | 100ms | ease-in | scale(0.97) active feedback |
| Category pill selection | 150ms | ease-out | Background color transition |
| Promo code error shake | 300ms | ease-in-out | Horizontal shake, 3px amplitude |

| Sold out badge | none | — | Static, no animation |

Rule: No animation in payment/checkout path exceeds 150ms. Stripe redirect is external.

## Accessibility Requirements

| Requirement | Implementation |
|-------------|----------------|
| Product images | alt text = product name; if no image, aria-label on placeholder |
| Cart icon | aria-label="Cart, {n} items" updating dynamically |
| Sold out product card | aria-disabled="true", remove click handler, add role="img" note |
| Cart drawer | role="dialog", aria-modal="true", focus trap when open |
| Quantity stepper | aria-label="Increase quantity" / "Decrease quantity" |
| Promo code error | aria-live="polite" on error message container |
| Category pills | role="radiogroup" + role="radio" pattern, keyboard navigable |
| All buttons | Minimum 44x44px touch target on mobile |
| Color contrast | Navy on white: 12.6:1 (AAA). Amber on white: 3.1:1 — amber used only on large text/buttons (AA large). |
| CartLineItem remove button | aria-label="Remove {product name} from cart" |
