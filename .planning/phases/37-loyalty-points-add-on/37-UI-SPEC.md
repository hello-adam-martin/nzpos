| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Compact element spacing, inner card padding |
| md | 16px | Default element spacing, form field gaps |
| lg | 24px | Section padding, card padding |
| xl | 32px | Layout gaps between sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Note: Grain texture and hairline separators are CSS-layer visual treatments (e.g., `border: 1px solid`) — not spacing tokens.

Exceptions:
- POS touch targets: minimum 44×44px (all interactive elements in POS flow — "Add Customer" button, search result rows, Apply Points button, Cancel button).
- Points redemption row in CartPanel: minimum 40px row height, consistent with existing CartLineItem rows.
- Customer lookup sheet width: 320px (w-80), matching DiscountSheet width.

## Interaction Contracts

### POS Customer Lookup Flow

1. "Add Customer" button appears in CartPanel below the cart items list, above PaymentMethodToggle. Small, secondary style (border button, not amber) when no customer is attached. Amber CTA style after customer is attached shows customer name ("Jane Smith").
2. Tap opens CustomerLookupSheet (slide-in from right, 150ms). Input auto-focuses.
3. Type-ahead: debounce 300ms, minimum 2 characters before search fires. Results list shows name + email + points balance per row. Rows are 44px minimum height.
4. "No match found" appears as inline text below empty results list, with "Create new customer" button (ghost style, full width, 44px).
5. Quick-add form: name field (required), email field (required), consent checkbox ("Customer has been informed about loyalty data collection" — must be checked to enable Save). Save button is navy, disabled until all fields valid + checkbox checked.
6. After customer attached: sheet closes, CartPanel shows LoyaltyRedemptionRow.
7. Staff can tap "Add Customer" again to change customer (replaces current attachment).
8. "Skip" link always visible at bottom of sheet — closes without attaching customer.

### POS Points Redemption

1. LoyaltyRedemptionRow shows: "{Name}: {N} pts ({$X.XX} available)" with amber "Apply" button.
2. Tap "Apply": row transitions to discount state showing "Points discount  -$X.XX" in amber text with a ghost "Remove" button (navy border). Haptic analogy: same visual weight as DiscountSheet applied state.
3. Partial redemption: staff can only apply full available balance (no partial points entry in POS — simplicity over flexibility per D-08 "no minimum threshold").
4. When points discount applied, CartSummary updates totals. Points are NOT earned on the discounted portion (D-09).

### Online Checkout Points Redemption

1. "Use your points" section appears below order summary, above payment section. Visible only when customer is logged in and has points balance > 0 and loyalty is active.
2. Displays: "You have 450 pts worth $4.50" with "Apply" button (amber).
3. When applied: button changes to "Remove" (ghost), negative line item appears in order summary: "Points discount  −$4.50".
4. Mirrors gift card negative line item pattern from Phase 35.

### Online Privacy Banner (D-11)

1. Info-tinted banner (blue #3B82F6 at 10% bg tint, blue border, blue icon) appears at top of account page on first visit after loyalty activates.
2. Copy: "You're now earning loyalty points! We track your purchase history to calculate your reward balance. [Learn more] [OK]"
3. "Learn more" is a text link to a brief inline explanation or privacy policy section.
4. "OK" dismisses and sets a `loyalty_banner_dismissed` flag in DB. Never shown again.
5. Duration: no auto-dismiss. User must explicitly tap OK.

### Admin Setup Gate (D-10)

1. After subscribing to Loyalty ($15/mo), admin loyalty page shows a warning banner: "Set up earn and redeem rates to activate loyalty points. Points will not accumulate until both rates are saved."
2. Warning color: #D97706 (amber warning), not destructive red.
3. Once both rates are saved (non-zero), banner disappears. Earning activates.

## Accessibility Requirements

- All interactive elements in POS flow: minimum 44×44px touch target (iPad use case).
- CustomerLookupSheet: `role="dialog"`, `aria-modal="true"`, `aria-label="Add customer"`. Focus trapped within sheet while open. Close on Escape key.
- Privacy banner: `role="banner"` or `role="alert"` with `aria-live="polite"`. Dismiss button has `aria-label="Dismiss loyalty notice"`.
- Consent checkbox in quick-add form: associated label via `htmlFor`/`id`. Cannot save without checked.
- Points balance: announced to screen readers as "{N} points, worth {$X.XX}".
- Error states: `role="alert"` on error paragraphs.
- Admin loyalty page: earn rate and redeem rate inputs have explicit `<label>` elements with `htmlFor`.

## Motion Contract

All animations follow DESIGN.md motion rules:

| Animation | Duration | Easing | Applied To |
|-----------|----------|--------|------------|
| CustomerLookupSheet open/close | 150ms | ease-out (open), ease-in (close) | translate-x-full ↔ translate-x-0 |
| LoyaltyRedemptionRow appear | 150ms | ease-out | opacity 0→1 + slide down |
| Applied state transition | 100ms | ease-out | color swap amber, label change |
| Privacy banner appear | 150ms | ease-out | opacity 0→1 |
| Privacy banner dismiss | 150ms | ease-in | opacity 1→0 |
| Search results list | 100ms | ease-out | opacity fade on result update |

POS rule enforced: nothing in the checkout path animates longer than 150ms.
